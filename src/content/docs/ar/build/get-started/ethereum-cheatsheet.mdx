### مقارنة مفصلة لمعايير الرموز المميزة

|                        | Solidity                                                               | Move (Aptos)                                                                 |
|------------------------|------------------------------------------------------------------------|------------------------------------------------------------------------------|
| **هيكل الرمز المميز** | كل رمز مميز هو عقده الخاص.                                             | كل رمز مميز هو `Coin` أو `FungibleAsset` مكتوب باستخدام عقد واحد قابل لإعادة الاستخدام. |
| **معيار الرمز المميز** | يجب الامتثال للمعايير مثل ERC20؛ قد تختلف التطبيقات.                   | واجهة موحدة وتطبيق لجميع الرموز المميزة.                                    |
| **تخزين الرصيد**      | تخزين الأرصدة في العقد باستخدام هيكل التعيين.                          | **الرصيد الموجه نحو الموارد**: يتم تخزين الأرصدة كمورد في حساب المستخدم. لا يمكن إنشاء الموارد بشكل تعسفي، مما يضمن سلامة قيمة الرمز المميز. |
| **آلية النقل**        | يمكن نقل الرموز المميزة دون إذن صريح من المتلقي.                       | باستثناء الحالات المحددة (مثل AptosCoin)، الرموز المميزة تتطلب عادةً سلطة `signer` الخاصة بالمتلقي للنقل. |

### مقارنة مفصلة بين EVM وMove VM

- **EVM**: معروف بمرونته وإرساله الديناميكي، مما يسمح بنطاق واسع من سلوكيات العقود الذكية. هذه المرونة، ومع ذلك، قد تؤدي إلى تعقيدات في التنفيذ المتوازي وعمليات الشبكة.
- **Move VM**: يركز على الأمان والكفاءة مع نهج أكثر تكاملاً بين الجهاز الافتراضي ولغة البرمجة. نموذج تخزين البيانات الخاص به يسمح بتوازن أفضل، وطريقة الإرسال الثابتة تعزز الأمان والقدرة على التنبؤ.

<br />

|                                 | EVM (Ethereum Virtual Machine)                                         | Move VM (Move Virtual Machine)                                     |
|---------------------------------|------------------------------------------------------------------------|-------------------------------------------------------------------|
| **تخزين البيانات**             | يتم تخزين البيانات في مساحة تخزين العقد الذكي.                        | يتم تخزين البيانات عبر العقود الذكية، حسابات المستخدمين، والكائنات. |
| **التوازي**                    | التنفيذ المتوازي محدود بسبب مساحة التخزين المشتركة.                   | يتيح المزيد من التنفيذ المتوازي بفضل تصميم التخزين المنقسم المرن.   |
| **تكامل الجهاز الافتراضي واللغة** | طبقات منفصلة لـ EVM ولغات العقود الذكية (مثل Solidity).                | تكامل سلس بين طبقة الجهاز الافتراضي ولغة Move، مع وظائف أصلية مكتوبة في Rust قابلة للتنفيذ في Move. |
| **عمليات الشبكة الحرجة**       | تنفيذ عمليات الشبكة قد يكون معقداً وغير مباشر.                        | العمليات الحرجة مثل إدارة مجموعة المدققين يتم تنفيذها محلياً في Move، مما يسمح بالتنفيذ المباشر. |
| **استدعاء الوظائف**            | الإرسال الديناميكي يسمح باستدعاءات العقود الذكية التعسفية.             | الإرسال الثابت يتماشى مع التركيز على الأمان والسلوك المتوقع.       |
| **أمان النوع**                 | أنواع العقود توفر مستوى من أمان النوع.                                | هياكل الوحدة والأنواع العامة في Move توفر أمان نوع قوي.             |
| **أمان المعاملة**              | يستخدم nonces لترتيب المعاملات وأمانها.                               | يستخدم أرقام تسلسلية لترتيب المعاملات وأمانها.                      |
| **تخزين مصدق**                 | نعم، مع تخزين العقد الذكي.                                            | نعم، مع الاستفادة من نموذج الموارد في Move.                        |
| **إمكانية الوصول إلى الكائن**  | الكائنات ليست قابلة للوصول عالمياً؛ مرتبطة بنطاق العقد الذكي.         | ضمان إمكانية الوصول العالمية للكائنات.                             |
