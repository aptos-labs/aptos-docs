Ένας λογαριασμός στο blockchain του Aptos αντιπροσωπεύει έλεγχο πρόσβασης σε ένα σύνολο περιουσιακών στοιχείων, συμπεριλαμβανομένου του νομίσματος on-chain και των NFTs. Στο Aptos, αυτά τα περιουσιακά στοιχεία αντιπροσωπεύονται από μια πρωτόγονη δομή της γλώσσας Move που ονομάζεται **resource** και τονίζει τόσο τον έλεγχο πρόσβασης όσο και τη σπανιότητα.

Κάθε λογαριασμός στο blockchain του Aptos αναγνωρίζεται από μια διεύθυνση λογαριασμού 32 bytes. Μπορείτε να χρησιμοποιήσετε την Υπηρεσία Ονομάτων Aptos στο [www.aptosnames.com](https://www.aptosnames.com/) για να εξασφαλίσετε domains .apt για κύριους λογαριασμούς ώστε να είναι αξιομνημόνευτοι και μοναδικοί.

Διαφορετικά από άλλα blockchains όπου οι λογαριασμοί και οι διευθύνσεις είναι σιωπηρές, οι λογαριασμοί στο Aptos είναι ρητοί και πρέπει να δημιουργηθούν πριν μπορέσουν να εκτελέσουν συναλλαγές. Ο λογαριασμός μπορεί να δημιουργηθεί ρητά ή σιωπηρά μεταφέροντας tokens Aptos (APT) εκεί. Δείτε την ενότητα [Δημιουργία λογαριασμού](#creating-an-account) για περισσότερες λεπτομέρειες. Κατά κάποιον τρόπο, αυτό είναι παρόμοιο με άλλες αλυσίδες όπου μια διεύθυνση πρέπει να λάβει κεφάλαια για gas πριν μπορέσει να στείλει συναλλαγές.

Οι ρητοί λογαριασμοί επιτρέπουν πρωτόγονες λειτουργίες που δεν είναι διαθέσιμες σε άλλα δίκτυα όπως:

- Εναλλαγή κλειδιού αυθεντικοποίησης. Το κλειδί αυθεντικοποίησης του λογαριασμού μπορεί να αλλάξει ώστε να ελέγχεται μέσω ενός διαφορετικού ιδιωτικού κλειδιού. Αυτό είναι παρόμοιο με την αλλαγή κωδικών πρόσβασης στον κόσμο του web2.
- Εγγενής υποστήριξη multisig. Οι λογαριασμοί στο Aptos υποστηρίζουν k-of-n multisig χρησιμοποιώντας τόσο τα σχήματα υπογραφής Ed25519 όσο και Secp256k1 ECDSA κατά την κατασκευή του κλειδιού αυθεντικοποίησης.

Υπάρχουν τρεις τύποι λογαριασμών στο Aptos:

- _Τυπικός λογαριασμός_ - Αυτός είναι ένας τυπικός λογαριασμός που αντιστοιχεί σε μια διεύθυνση με ένα αντίστοιχο ζεύγος δημόσιων/ιδιωτικών κλειδιών.
- [_Λογαριασμός πόρων_](/el/build/smart-contracts/resource-accounts) - Ένας αυτόνομος λογαριασμός χωρίς αντίστοιχο ιδιωτικό κλειδί που χρησιμοποιείται από προγραμματιστές για την αποθήκευση πόρων ή τη δημοσίευση modules on-chain.
- [_Αντικείμενο_](/el/build/smart-contracts/objects) - Ένα πολύπλοκο σύνολο πόρων που αποθηκεύονται σε μια μόνο διεύθυνση αντιπροσωπεύοντας μια μόνο οντότητα.
