Para Linux, la forma más fácil de instalar la herramienta CLI de Aptos es a través de un script de shell, aunque si eso no funciona, también puedes instalar manualmente descargando binarios precompilados. El enfoque de binarios precompilados no se recomienda generalmente ya que actualizar es muy manual.

# Instalar via Script

<Steps>
  1. En la terminal, usa uno de los siguientes comandos:

     ```shellscript filename="Terminal"
     curl -fsSL "https://aptos.dev/scripts/install_cli.sh" | sh
     ```

     O usa el comando `wget` equivalente:

     ```shellscript filename="Terminal"
     wget -qO- "https://aptos.dev/scripts/install_cli.sh" | sh
     ```

     <Aside type="caution">
       Si estás obteniendo errores de `Instrucción ilegal` al ejecutar el CLI, puede ser debido a que tu CPU no soporta instrucciones SIMD.
       Específicamente para procesadores más antiguos que no soportan SIMD o contenedores Docker de Ubuntu x86\_64 en Macs ARM, es posible que necesites ejecutar el siguiente comando en su lugar para omitir las instrucciones SIMD:

       ```shellscript filename="Terminal"
         curl -fsSL "https://aptos.dev/scripts/install_cli.sh" | sh -s -- --generic-linux
       ```
     </Aside>

  2. (Opcional) Puede ser útil agregar el CLI de Aptos a una carpeta en tu PATH, o agregarlo a tu PATH directamente.

     - Los pasos para agregar una carpeta a tu PATH dependen del shell.
     - Puedes ejecutar `echo $SHELL` para imprimir el shell predeterminado de tu máquina, luego buscar en Google los pasos específicos para agregar una carpeta a tu PATH para ese shell.

  3. Verifica que el script esté instalado abriendo una nueva terminal y ejecutando aptos help

     - Deberías ver una lista de comandos que puedes ejecutar usando el CLI.
     - En el futuro, este es un recurso útil para aprender exactamente cómo funciona cada comando.
</Steps>

<Aside type="note" emoji="ℹ️">
  Si te gustaría actualizar el CLI de Aptos a la versión más reciente, puedes ejecutar `aptos update`.
</Aside>

# Instalar via Gestor de Paquetes (Opcional)

<Aside type="note">
  Al instalar Aptos a través de un gestor de paquetes, por favor actualízalo a través del mismo gestor de paquetes en el futuro.
</Aside>

### Arch Linux

#### Instalar via AUR (Arch User Repository)

```shellscript filename="Terminal"
git clone https://aur.archlinux.org/aptos-bin.git
cd aptos-bin
makepkg -si
```

o usa un helper de AUR como `yay`:

```shellscript filename="Terminal"
yay -S aptos-bin
```

# Instalar via Binarios Precompilados (Método de Respaldo)

<Steps>
  1. Ve a la página de lanzamientos de Aptos CLI en GitHub.

  2. Haz clic en el menú expandible "Assets" para el lanzamiento más reciente para ver los binarios precompilados.

  3. Descarga el archivo zip para Linux.

     1. Tendrá un nombre como: `aptos-cli-<version>-Linux-x86_64.zip` o `aptos-cli-<version>-Linux-aarch64.zip`.
     2. Asegúrate de elegir el archivo zip correcto para la arquitectura de tu computadora (x86\_64 para Intel / AMD o aarch64 para ARM).
     3. Probablemente tendrás que descartar advertencias de que este es un archivo sospechoso al descargarlo.

  4. Descomprime el archivo descargado.

  5. Mueve el archivo binario de Aptos extraído a tu carpeta preferida.

  6. Abre una terminal y navega a tu carpeta preferida.

  7. Haz que ~/aptos sea ejecutable ejecutando chmod +x ~/aptos.

  8. Verifica que esta versión instalada funcione ejecutando ~/aptos help.

     Deberías ver instrucciones sobre cómo usar todos los comandos del CLI. Estos pueden ser útiles en el futuro cuando estés tratando de entender cómo usar comandos específicos.

  9. (Opcional) Puede ser útil agregar el CLI de Aptos a una carpeta en tu PATH, o agregarlo a tu PATH directamente.

     - Los pasos para agregar una carpeta a tu PATH dependen del shell.
     - Puedes ejecutar `echo $SHELL` para imprimir el shell predeterminado de tu máquina, luego buscar en Google los pasos específicos para agregar una carpeta a tu PATH para ese shell.
</Steps>

<Aside type="note" emoji="ℹ️">
  Al usar el método de binarios precompilados, puedes actualizar el CLI de Aptos eliminando tu instalación existente, luego siguiendo los pasos de instalación nuevamente.
</Aside>
