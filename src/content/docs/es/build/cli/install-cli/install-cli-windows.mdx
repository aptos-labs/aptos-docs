---
title: "Instalar la CLI de Aptos en Windows"
---

Para Windows, la forma más fácil de instalar la herramienta CLI de Aptos es mediante script de PowerShell. Si eso no funciona, también puedes instalar manualmente mediante binarios precompilados. El enfoque de binarios precompilados no se recomienda generalmente ya que las actualizaciones son muy manuales.

# Instalar mediante Script de PowerShell

<Steps>
  1. En PowerShell, ejecuta el script de instalación:

     ```powershell filename="Terminal"
     Set-ExecutionPolicy RemoteSigned -Scope CurrentUser; iwr https://aptos.dev/scripts/install_cli.ps1 | iex
     ```

  2. Verifica que el script esté instalado abriendo una nueva terminal y ejecutando aptos help.

     - Deberías ver una lista de comandos que puedes ejecutar usando la CLI.
     - En el futuro, este es un recurso útil para aprender exactamente cómo funciona cada comando.
</Steps>

<Aside type="note" emoji="ℹ️">
  Si deseas actualizar la CLI de Aptos a la última versión mediante script, puedes ejecutar `aptos update`.
</Aside>

# Instalar mediante Gestor de Paquetes (Opcional)

<Aside type="note">
  Al instalar Aptos mediante un gestor de paquetes, por favor actualízalo a través del mismo gestor de paquetes en el futuro.
</Aside>

### Si tienes [Scoop](https://scoop.sh/) instalado, puedes ejecutar el siguiente comando para instalar la CLI de Aptos:

```powershell filename="Terminal"
scoop install https://aptos.dev/scoop/aptos.json
```

### Si tienes [Chocolatey](https://chocolatey.org/) instalado, puedes ejecutar el siguiente comando para instalar la CLI de Aptos:

```powershell filename="Terminal"
choco install aptos
```

### Si tienes [winget](https://winget.run/) instalado, puedes ejecutar el siguiente comando para instalar la CLI de Aptos:

```powershell filename="Terminal"
winget install aptos
```

# Instalar mediante Binarios Precompilados (Método de Respaldo)

<Steps>
  1. Ve a la [página de releases de GitHub](https://github.com/aptos-labs/aptos-core/releases).

  2. Expande "Assets" para ver los binarios precompilados.

  3. Descarga el archivo zip para Windows.

     - Tendrá un nombre como: `aptos-cli-<version>-Windows-x86_64.zip`
     - Es probable que tengas que descartar advertencias de que este es un archivo sospechoso al descargarlo.

  4. Descomprime el archivo descargado.

     - Mueve el archivo a la carpeta que quieras llamar `aptos` en el futuro.

  5. Haz clic derecho, luego copia la ruta al ejecutable.

     Ej. `C:\Users\<username>\Downloads\aptos-cli-3.1.0-Windows-x86_64\aptos.exe`.

     <Aside type="note" emoji="ℹ️">
       Es posible que quieras agregar esta ruta a tu variable de entorno PATH para simplificar la llamada a la CLI de Aptos en el futuro.
     </Aside>

  6. Abre PowerShell a través del Menú Inicio.

  7. Verifica la instalación ejecutando el comando de ayuda.

     Usa la ruta que copiaste anteriormente para llamar a la CLI de Aptos.
     Ej. `C:\Users\<username>\Downloads\aptos-cli-3.1.0-Windows-x86_64\aptos.exe help`.
</Steps>

<Aside type="note" emoji="ℹ️">
  Al instalar con binarios precompilados, puedes actualizar la CLI de Aptos eliminando tu instalación existente, luego siguiendo los pasos de instalación nuevamente.
</Aside>

<Aside type="caution">
  Si ninguno de los métodos anteriores funciona, tendrás que compilar la CLI desde el código fuente siguiendo estos pasos: [Instalar Versiones Específicas de la CLI de Aptos (Avanzado)](/build/cli/install-cli/install-cli-specific-version)
</Aside>
