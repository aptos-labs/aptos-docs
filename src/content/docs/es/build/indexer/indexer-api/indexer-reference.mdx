La API del Indexador te permite acceder a datos enriquecidos sobre tokens, activos fungibles y cuentas on-chain usando consultas GraphQL. **Puedes acceder a ella [aquí](/es/build/indexer/indexer-api).**

Para consultas comunes, revisa la barra lateral para ver ejemplos. Al construir las tuyas, esta guía de referencia te ayudará a determinar qué tablas son más relevantes y cómo formatear tus consultas.

<Aside type="caution">
  Antes de depender de una tabla para servicios en producción, revisa al final de esta página si esa tabla está deprecada. De ser así, usa la sección de notas para orientación sobre cómo migrar a una tabla no deprecada.
</Aside>

<Aside type="note">
  Si estás buscando una tabla con el sufijo `_by_pk`, busca el nombre de la tabla sin ese sufijo. Las tablas `_by_pk` se generan automáticamente para permitir consultas por clave primaria.
</Aside>

# Referencia de Tablas del Indexador

<Aside type="note">
  ¡Recuerda usar Ctrl + F para encontrar la tabla que te interesa! En caso de duda, también puedes consultar las tablas de Hasura enlazadas en la página de [Acceso a la API del Indexador](/es/build/indexer/indexer-api) para ver ejemplos de los datos.
</Aside>

## Filtrado (con cláusulas `where`)

Para asegurar que tus consultas filtren datos eficientemente, revisa los índices disponibles para cada tabla.
Algunos índices son índices B-tree compuestos, lo que significa que constan de múltiples columnas.
Los índices B-tree están ordenados y funcionan de manera óptima cuando las consultas utilizan un prefijo más a la izquierda de las columnas indexadas.
