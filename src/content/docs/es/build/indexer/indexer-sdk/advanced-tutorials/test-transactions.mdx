El Framework de Pruebas del Indexer de Aptos proporciona dos formas de generar transacciones de prueba: **importando Transacciones de la Red** y **escribiendo Scripts Move**. Ambos enfoques son adecuados para escenarios específicos basados en tus requisitos de desarrollo y pruebas, permitiéndote probar cómo tu sistema maneja varios tipos de transacciones.

## Cuándo Importar transacciones

Las transacciones importadas se usan principalmente para validar lógica de procesador o integridad de base de datos reproduciendo transacciones de redes en vivo.

## Cuándo Usar **Script Move** para generar transacciones

Las transacciones con script se usan principalmente para crear y probar escenarios de transacciones que aún no están en vivo en la red. En la mayoría de los casos, deberías usar importación de transacciones para probar tu lógica de procesador.

## Resumen

La herramienta Aptos-indexer-transaction-generator es una herramienta esencial en el Framework de Pruebas del Indexer de Aptos. Importa transacciones para reproducir y analizar transacciones del mundo real, mientras que generar transacciones con **Scripts Move** es mejor para probar nuevos AIPs que pueden impactar la lógica de procesamiento. Elige el método que se alinee con tus objetivos de prueba para asegurar un proceso de validación integral.
