Si estás creando un procesador personalizado desde cero, recomendamos seguir la [Guía de Inicio Rápido](/es/build/indexer/indexer-sdk/quickstart).
La guía de inicio rápido proporciona un procesador plantilla e incluye toda esta configuración.

Si estás migrando un procesador existente al SDK de Indexer, sigue los pasos abajo.

Agrega `aptos-indexer-processor-sdk` a tu `Cargo.toml`.

```toml
[dependencies]
aptos-indexer-processor-sdk = { git = "https://github.com/aptos-labs/aptos-indexer-processor-sdk.git", rev = "aptos-indexer-processor-sdk-v1.0.0" }
```

`aptos-indexer-processor-sdk` incluye las siguientes características:

1. `postgres_full` - Capa de interfaz para integrar Postgres con tu procesador.
2. `testing_framework` - Un framework de testing e2e para probar procesadores. Si quieres escribir pruebas para tu procesador, agrega esta característica al crate.
