La Serialización Canónica Binaria (BCS) es el formato de serialización usado en la blockchain de Aptos.
Es un formato de serialización binario canónico no autodescriptivo
que se usa para serializar estructuras de datos. BCS se usa para serializar todos los datos
en la cadena, proporcionar respuestas binarias en la API REST, y codificar argumentos de entrada
a transacciones.

## Resumen

Debido a que BCS no es un formato autodescriptivo, el lector debe conocer el formato de
los bytes de antemano.

## Tipos Primitivos

Se soportan enteros sin signo de 8-bit, 16-bit, 32-bit, 64-bit, 128-bit y 256-bit.
Se serializan en orden de bytes little-endian.

### Bool (booleano)

Los booleanos se serializan como un solo byte. `true` se serializa como `0x01` y
`false` se serializa como `0x00`. Todos los otros valores son inválidos.

| Valor   | Bytes  |
| ------- | ------ |
| `true`  | `0x01` |
| `false` | `0x00` |

### Uleb128 (entero de 128 bits sin signo de longitud variable)

Los enteros sin signo de 128 bits de longitud variable se serializan como un número variable de bytes. El bit más significativo de cada byte indica si hay más bytes por leer. Los 7 bits restantes se usan para almacenar el valor.

Esto se usa comúnmente para longitudes variables de vectores o para enums.

### Sequence y FixedSequence

Las secuencias se serializan como un vector de longitud variable de un ítem. La longitud del vector se serializa como un Uleb128 seguido de los elementos. Las FixedSequences se serializan sin el byte inicial de tamaño. El lector debe conocer el número de bytes antes de deserializar.

<Tabs>
  <TabItem label="Move">
    ```move filename="vector.move"
    #[test_only]
    module 0x42::example {
      use std::bcs;
      use std::from_bcs;

      #[test]

        // Serializar
        let val = vector[1u8, 2u8, 3u8];
        let bytes = bcs::to_bytes(&val);
        assert!(bytes == vector[3, 1, 2, 3]);

        // Deserializar, solo soporta bytes por ahora
        let val_des = from_bcs::to_bytes(bytes);
        assert!(val_des == vector[1, 2, 3]);
      }
    }
    ```
  </TabItem>

  <TabItem label="Rust">

    // Deserializar

    const val = des.deserializeVector((d) => d.deserializeU8());

  </TabItem>

  <TabItem label="Go">

      "github.com/aptos-labs/aptos-go-sdk"
    )

      // Serializar
      ser := bcs.Serializer{}
      ser.SerializeVector([]uint8{1, 2, 3}, func(s *bcs.Serializer, item uint8) {
        s.U8(item)
      })
      bytes := ser.ToBytes()
      bytes == []byte{3, 1, 2, 3}

      // Deserializar
      des := bcs.NewDeserializer(bytes)
      val := des.DeserializeVector(func(d *bcs.Deserializer) uint8 {
        return d.U8()
      })
      val == []uint8{1, 2, 3}
    }

## Tipos Complejos

### String

Las cadenas se serializan como un vector de bytes; sin embargo, los bytes están codificados en UTF-8.

```rust

];

```

### AccountAddress

AccountAddress se serializa como un vector fijo de 32 bytes.

```
@0x1 => [0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x01]
```

### Struct

Los structs se serializan como un conjunto ordenado de campos. Los campos se serializan en el orden en que se definen en el struct.

```
struct Color {
  r: u8 = 1,
  g: u8 = 2,
  b: u8 = 3,
} => [0x01, 0x02, 0x03]
```

### Option

Las opciones se serializan como un solo byte para indicar si están presentes. Si la opción es `None`, el byte es `0x00`. Si la opción es `Some`, el byte es `0x01` seguido del valor serializado.

```rust
let some_data: Option<u8> = Some(8);
assert_eq!(to_bytes(&some_data)?, vec![1, 8]);

let no_data: Option<u8> = None;
assert_eq!(to_bytes(&no_data)?, vec![0]);
```

### Enum

Los enums se serializan como un uleb128 para determinar qué variante se usa. Al tamaño le sigue el valor serializado de la variante.

```rust
#[derive(Serialize)]
enum E {
    Variant0(u16),
    Variant1(u8),
    Variant2(String),
}

let v0 = E::Variant0(8000);
let v1 = E::Variant1(255);
let v2 = E::Variant2("e".to_owned());

assert_eq!(to_bytes(&v0)?, vec![0, 0x40, 0x1F]);
assert_eq!(to_bytes(&v1)?, vec![1, 0xFF]);
assert_eq!(to_bytes(&v2)?, vec![2, 1, b'e']);
```

### Maps

Los mapas se almacenan como una secuencia de tuplas. La longitud del mapa se serializa como un Uleb128 seguido de pares clave-valor repetidos.

```rust
let mut map = HashMap::new();
map.insert(b'e', b'f');
map.insert(b'a', b'b');
map.insert(b'c', b'd');

let expecting = vec![(b'a', b'b'), (b'c', b'd'), (b'e', b'f')];

assert_eq!(to_bytes(&map)?, to_bytes(&expecting)?);
```
