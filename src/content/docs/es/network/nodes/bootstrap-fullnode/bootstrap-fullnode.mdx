Este documento describe cómo inicializar un nodo Aptos rápidamente usando un snapshot. Esto se puede hacer en todos los tipos de nodos,
incluyendo validators, VFNs y PFNs.

<Aside type="caution">
  **Snapshots de indexer**<br />
  Los snapshots proporcionados por la comunidad no proporcionan soporte de indexer. Si estás inicializando un nodo indexer,
  necesitarás hacerlo usando un [backup](/network/nodes/bootstrap-fullnode/aptos-db-restore).
</Aside>

Aunque puedes inicializar un nuevo nodo usando [sincronización de estado](/network/nodes/configure/state-sync), este podría no ser el enfoque más rápido después de que la red
haya estado ejecutándose por un tiempo; puede tomar demasiado tiempo, o no podrá obtener todos los datos requeridos ya que
otros nodos pueden haber podado ya su historial del ledger. Para evitar esto, puedes inicializar tu nodo usando un
snapshot existente, que es simplemente una copia de los datos de almacenamiento de un nodo existente.

<Aside type="caution">
  **Snapshots de mainnet**<br />
  No se recomienda usar snapshots para inicializar nodos en **mainnet**. Esto es porque los snapshots
  no son verificados por el software de Aptos. Como resultado, el snapshot puede ser inválido o contener datos incorrectos. Para prevenir
  preocupaciones de seguridad, recomendamos usar snapshots solo en entornos de prueba, ej., **devnet** y **testnet**.

  Si debes inicializar un nodo usando un snapshot de **mainnet**, deberías generar el snapshot tú mismo
  u obtenerlo de una fuente confiable. También deberías verificar la integridad y autenticidad del snapshot antes de usarlo
  (ej., vía firmas criptográficas y checksums).
</Aside>

## Encontrar un snapshot existente

Hay varios snapshots que pueden descargarse de diferentes miembros de la comunidad de Aptos. Estos incluyen:

- BWareLabs (Testnet y Mainnet): [Snapshots de Nodos Aptos BWareLabs](https://bwarelabs.com/snapshots/aptos)

<Aside type="note">
  **Preguntas sobre datos de snapshot** <br />
  Dependiendo de cómo se construya y comprima el snapshot, los archivos de snapshot pueden ser de diferentes tamaños. Si tienes
  preguntas sobre los datos del snapshot, o encuentras problemas, por favor contacta a los miembros de la comunidad de Aptos directamente
  vía el canal [#node-support](https://discord.com/channels/945856774056083548/953421979136962560) en [Discord de Aptos](https://discord.gg/aptosnetwork).
</Aside>

## Usar un snapshot

Para usar un snapshot, simplemente descarga y copia los archivos a la ubicación de la base de datos de almacenamiento para tu nodo.
Esta ubicación se puede encontrar y actualizar en el archivo de configuración `yaml` del fullnode bajo `data_dir`.
Ver el tutorial de ejemplo ([Ejecutar un PFN](/network/nodes/full-node/deployments)) para cómo configurar el directorio de datos.
