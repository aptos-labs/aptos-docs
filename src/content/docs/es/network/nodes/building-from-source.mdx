## Sistemas operativos soportados

Aptos puede construirse en varios sistemas operativos, incluyendo Linux, macOS, y
Windows. Aptos se prueba extensivamente en Linux y macOS, y menos en Windows.
Aquí están las versiones que usamos:

- Linux - Ubuntu versión 20.04 y 22.04
- macOS - macOS Monterey y posterior
- Microsoft Windows - Windows 10, 11 y Windows Server 2022+

## Clonar el repo aptos-core

1. Instala [Git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git).
   Git es requerido para clonar el repo aptos-core, y necesitará ser instalado
   antes de continuar. Puedes instalarlo con las instrucciones en el sitio web oficial de
   [Git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git).

2. Clona el repositorio Aptos. Para clonar el repositorio (repo) Aptos, primero
   necesitas abrir un prompt de línea de comandos (Terminal en macOS / Linux, PowerShell en
   Windows). Luego ejecuta el siguiente comando para clonar el repositorio Git desde GitHub.

   ```shellscript filename="Terminal"
   git clone https://github.com/aptos-labs/aptos-core.git
   ```

3. Ahora vamos al directorio recién creado `aptos-core` _cambiando directorio_ o haciendo `cd`:
   ```shellscript filename="Terminal"
   cd aptos-core
   ```

## Configurar dependencias de construcción

Prepara tu entorno de desarrollador instalando las dependencias necesarias para construir, probar e inspeccionar Aptos Core.
No importa tu mecanismo seleccionado para instalar estas dependencias, **es imperativo que mantengas toda tu cadena de herramientas actualizada**.
Si encuentras problemas después, actualiza todos los paquetes e intenta de nuevo.

<details>
  <summary>macOS</summary>

  **> Usando el script automatizado**

  1. Asegúrate de tener el gestor de paquetes `brew` instalado: https://brew.sh/
  2. Ejecuta el script de configuración de dev para preparar tu entorno:

  ```shellscript filename="Terminal"
  ./scripts/dev_setup.sh
  ```

  3. Actualiza tu entorno de shell actual:

  ```shellscript filename="Terminal"
  source ~/.cargo/env
  ```

  <Aside type="note">
    Puedes ver las opciones disponibles para el script ejecutando

    ```shellscript filename="Terminal"
    ./scripts/dev_setup.sh --help
    ```
  </Aside>

  **> Instalación manual de dependencias**

  Si el script anterior no funciona para ti, puedes instalar estas manualmente, pero **no se recomienda**.

  1. [Rust](https://www.rust-lang.org/tools/install)
  2. [CMake](https://cmake.org/download/)
  3. [LLVM](https://releases.llvm.org/)
  4. [LLD](https://lld.llvm.org/)
</details>

<details>

  1. [Rust](https://www.rust-lang.org/tools/install).
  2. [CMake](https://cmake.org/download/).
  3. [LLVM](https://releases.llvm.org/).

<details>
  <summary>Windows</summary>

  **> Usando el script automatizado**

  1. Abre una terminal PowerShell como administrador.
  2. Ejecuta el script de configuración de dev para preparar tu entorno:

  ```powershell filename="Terminal"
  PowerShell -ExecutionPolicy Bypass -File ./scripts/windows_dev_setup.ps1
  ```

  3. Abre una nueva terminal PowerShell después de instalar todas las dependencias

  **> Instalación manual de dependencias**

  1. Instala [Rust](https://www.rust-lang.org/tools/install).
  2. Instala [LLVM](https://releases.llvm.org/). Visita su repositorio GitHub para el [último release precompilado](https://github.com/llvm/llvm-project/releases/tag/llvmorg-15.0.7).
  3. Instala [Microsoft Visual Studio Build Tools para Windows](https://visualstudio.microsoft.com/downloads/#build-tools-for-visual-studio-2022). Durante la configuración, selecciona "Desktop development with C++" y tres opciones adicionales: MSVC C++ build tools, Windows 10/11 SDK, y C++ CMake tools for Windows.
  4. Si estás en Windows ARM, instala [Visual Studio](https://visualstudio.microsoft.com/vs).
  5. Si no se instaló ya durante la instalación de Visual Studio/Build Tools, instala [CMake](https://cmake.org/download/).
  6. Abre una nueva terminal PowerShell después de instalar todas las dependencias
</details>

## Construir Aptos

La verificación más simple de que tienes un entorno funcionando es construir todo y ejecutar las pruebas.

```shellscript filename="Terminal"
cargo build
cargo test -- --skip prover
```

Si ya has instalado las Herramientas del Move Prover arriba entonces no necesitas
omitir las pruebas del prover. Para instalar el prover (opcional), sigue la [guía del Prover](/build/smart-contracts/prover)

Otra documentación de herramientas específicas tiene patrones recomendados para `cargo build` y `cargo run`
