Las siguientes guías proporcionan instrucciones paso a paso para desplegar un PFN en las redes de Aptos.

<Aside type="note">
  **¿Tengo que ejecutar un PFN?**<br />
  Si no deseas ejecutar un PFN, pero aún quieres interactuar con la blockchain de Aptos, puedes usar la API REST
  proporcionada por los PFNs de Aptos Labs (ver [APIs de Aptos](/build/apis)). Nota, sin embargo, que los PFNs proporcionados por Aptos Labs tienen límites de tasa, que pueden impedir tu desarrollo. Puedes obtener límites de tasa más altos creando una cuenta de [Geomi](https://geomi.dev/) y adjuntando un método de pago. También puedes desplegar tu propio PFN
  y sincronizar con la blockchain de Aptos directamente, o usar un proveedor de infraestructura de nodos diferente.
</Aside>

Selecciona la guía para el método de despliegue que prefieras:

<Aside type="caution">
  **Elegir una red**<br />
  Estas guías por defecto despliegan un PFN en la red mainnet de Aptos, pero pueden usarse fácilmente para hacer
  lo mismo en otras redes (ej., red devnet o testnet). Para hacerlo, cambia a la rama de red deseada
  y usa los archivos de nodo `genesis.blob` y `waypoint.txt` para las ramas respectivas:
  [mainnet](/network/nodes/configure/node-files-all-networks/node-files-mainnet),
  [devnet](/network/nodes/configure/node-files-all-networks/node-files-devnet), y
  [testnet](/network/nodes/configure/node-files-all-networks/node-files-testnet).
</Aside>
