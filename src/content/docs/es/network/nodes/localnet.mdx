Si quieres desarrollar y probar tus aplicaciones en la blockchain de Aptos, puedes ejecutar una
red local (**localnet**). Esta red se ejecutará en tu máquina local, y no estará
conectada a ninguna otra red de Aptos. Ejecutar una localnet es la forma más fácil de probar y
desarrollar aplicaciones de Aptos, y hacer cambios al software del nodo Aptos.

## Pasos Recomendados

La mayoría de desarrolladores deberían usar el CLI de Aptos para ejecutar una red de desarrollo local. Esta es la forma más simple
y fácil de comenzar. Proporciona un despliegue local que funciona igual que una red de producción,
incluyendo la [API del Nodo](/es/build/apis), [Stream de Transacciones](/es/build/indexer/txn-stream), [API del Indexer](/es/build/indexer) y [Faucet](/es/build/apis/faucet-api).

Para comenzar, sigue esta guía: [Localnet usando el CLI de Aptos](/es/network/nodes/localnet/local-development-network)

## Pasos Especializados

Si estás desarrollando el software central del nodo Aptos, o tienes necesidades de pruebas más complejas, considera las
guías abajo. La primera guía proporciona una forma de ejecutar una localnet con un solo nodo validator, y la
segunda guía proporciona una forma de ejecutar una localnet con múltiples nodos validator, y validator
fullnodes (VFNs).

Para ejecutar una localnet con un solo nodo validator, directamente desde el código fuente `aptos-core`, sigue esta guía:

Para ejecutar una localnet con múltiples nodos validator y validator fullnodes (VFNs), sigue esta guía:
