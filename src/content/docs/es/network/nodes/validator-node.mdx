Para participar en el protocolo de consenso de Aptos, debes desplegar y ejecutar un nodo validador y (opcionalmente) un nodo completo de validador (VFN). Este documento proporciona una visión general de alto nivel de los pasos importantes requeridos para desplegar ambos tipos de nodos.

<Aside type="note">
  Aunque los VFN no son requeridos para participar en consenso, es altamente recomendado que cada operador de validador ejecute un VFN.
  Esto es porque los VFN son los únicos puntos de entrada y salida de datos de blockchain para el ecosistema. Tener muchos VFN
  en la red ayuda a mejorar la confiabilidad, aumentar la disponibilidad de datos y proporcionar acceso de alta calidad a la blockchain al público.
</Aside>

Antes de inicializar un pool de staking o pool de delegación, lee sobre [Staking](/es/network/blockchain/staking) y [Staking Delegado](/es/network/blockchain/delegated-staking) para aprender las diferencias entre los tipos de pools de stake. Ten en cuenta que una vez que se ha creado un pool de stake, no puede cambiarse a un pool de delegación o viceversa.

Usa los documentos dentro de esta sección para ejecutar un validador de Aptos y un VFN. A alto nivel, el proceso es el siguiente:

<Steps>
  1. Entender los requisitos y tipos de despliegue.

     Lee los requisitos del nodo y selecciona un método de despliegue (ej., en premisas o servicios en la nube).
     Comienza leyendo los requisitos del nodo para conocer los recursos de computación, memoria, red y almacenamiento que necesitas.
     Además, selecciona un método para desplegar tus nodos, es decir, usa Kubernetes gestionado en la nube, Docker, o código fuente.

  2. Generar identidades para tus nodos.

     Crea tus pares de claves públicas/privadas y direcciones de cuenta para el validador y VFN.
     ¡Recuerda mantener tus claves privadas confidenciales!

  3. Configurar tu validador y VFN.

     Configura tus nodos para usar las claves e identidades generadas.
     Usando archivos YAML, configura tus nodos con las claves e identidades generadas en el paso anterior.
     Esto es requerido para permitir que tus nodos se conecten a otros nodos (es decir, pares) de forma segura.

  4. Descargar genesis y un waypoint para tus nodos.

     Arranca los nodos con un genesis y waypoint, y prepáralos para el inicio.
     Con los nodos configurados correctamente, instala los binarios y descarga el blob de genesis y archivos de waypoint.
     Estos darán a tus nodos la información que necesitan para comenzar a sincronizar con otros pares.

  5. Unirse al conjunto de validadores y comenzar a participar.

     Inicializa el pool de staking, únete al conjunto de validadores y comienza a validar para ganar recompensas.
     Antes de que otros pares acepten conexiones de tus nodos, necesitarás unirte al conjunto de validadores.
     Para hacer esto, debes inicializar un pool de staking y delegar a operadores y votantes.
     Una vez que tu pool de staking haya sido configurado, puedes unirte al conjunto de validadores.
     En este punto tus nodos comenzarán a sincronizar con la red y tu validador podrá comenzar a participar en consenso.
     Aquí es cuando puedes comenzar a ganar recompensas.
</Steps>

## Consideraciones Importantes

### Requisitos de Hardware

- **CPU**: Mínimo 8 núcleos, recomendado 16+ núcleos
- **RAM**: Mínimo 32GB, recomendado 64GB+
- **Almacenamiento**: SSD de alta velocidad con al menos 2TB
- **Red**: Conexión estable con ancho de banda alto

### Seguridad

- Mantén las claves privadas seguras y respaldadas
- Usa firewalls y configuraciones de red apropiadas
- Monitorea regularmente la salud y seguridad del nodo
- Implementa procedimientos de respuesta a incidentes

### Monitoreo y Mantenimiento

- Configura alertas para métricas críticas del nodo
- Mantén el software actualizado con las últimas versiones
- Monitorea el rendimiento y la participación en consenso
- Planifica para actualizaciones de red y mantenimiento
