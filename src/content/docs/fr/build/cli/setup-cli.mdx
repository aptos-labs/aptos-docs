Si vous utilisez la CLI pour essayer des choses sur la chaîne, vous devrez configurer le réseau, le faucet et les identifiants que vous voulez que la CLI utilise.

Cela rend l'utilisation de la CLI plus facile et plus sécurisée car vous ne serez pas forcé de copier à plusieurs reprises des adresses ou des clés privées.

<Aside type="caution">
  Si vous devez encore installer la CLI, suivez [ces étapes](/build/cli/install-cli/install-cli-specific-version).
</Aside>

1. Exécutez `aptos init` et suivez les instructions dans la ligne de commande.

<Aside type="note" emoji="ℹ️">
  Pour utiliser les paramètres par défaut, vous pouvez ne fournir aucune entrée et simplement appuyer sur "Entrée". Par exemple :
</Aside>

```shellscript filename="Terminal"
aptos init
```

## (Optionnel) Création de Configurations Nommées (Profils)

Pour tester des scénarios plus compliqués, vous voudrez souvent plusieurs comptes sur la chaîne. Une façon de faire cela est de créer une configuration nommée que nous appelons un profil.

Pour créer un profil, exécutez `aptos init --profile <name_of_profile>`. La configuration que vous générez sera utilisable lors de l'appel de commandes CLI en tant que remplacements pour les arguments.

Par exemple :

```shellscript filename="Terminal"
aptos init --profile bob
```

```shellscript filename="Terminal"
aptos account fund-with-faucet --profile bob
```


## (Optionnel) Configuration de l'Auto-Complétion Shell

Une fonctionnalité de qualité de vie que vous pouvez activer est l'auto-complétion shell.

1. Déterminez quel shell vous utilisez (vous pouvez exécuter `echo $SHELL` si vous n'êtes pas sûr).
2. Cherchez où vont les fichiers de configuration pour les complétions shell pour ce shell (cela varie d'un shell à l'autre). Les shells pris en charge sont `[bash, zsh, fish, PowerShell, elvish]`.
3. Exécutez la commande suivante avec votre shell spécifique et le fichier de sortie pour les complétions utilisant votre shell :

```shellscript filename="Terminal"
aptos config generate-shell-completions --shell <YOUR_SHELL_HERE> --output-file <OUTPUT_DESTINATION_FOR_YOUR_SHELL>
```

Exemple de commande pour [`oh my zsh`](https://ohmyz.sh/) :

```shellscript filename="Terminal"
aptos config generate-shell-completions --shell zsh --output-file ~/.oh-my-zsh/completions/_aptos
```

## (Optionnel) Configuration Globale

Par défaut, la CLI cherchera une configuration dans `.aptos/config.yaml` dans chaque répertoire d'espace de travail. Si vous voulez utiliser une configuration partagée pour tous les espaces de travail, vous pouvez suivre ces étapes :

1. Créez un dossier dans votre répertoire personnel appelé `.aptos` (afin qu'il ait le chemin `~/.aptos`).
2. Créez un fichier yaml à l'intérieur de `.aptos` appelé `global_config.yaml`.
3. Exécutez la commande :

```shellscript filename="Terminal"
aptos config set-global-config --config-type global
```

Vous devriez voir :

```json
{
  "Result": {
    "config_type": "Global"
  }
}
```
