import { CardGrid, LinkCard, Steps, TabItem, Tabs } from '@astrojs/starlight/components';

Voici une façon simple de configurer votre environnement selon le type de développement.

<Tabs>
  {/* Frontend */}

  <TabItem label="Frontend">
    <Steps>
      1. Initialiser le Projet Frontend

         Voici quelques exemples de choix populaires :

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. Installer @aptos-labs/ts-sdk

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. Configurer TS SDK

         <CardGrid>
           <LinkCard
             href="/build/sdks/ts-sdk/quickstart"
             title="Démarrage Rapide TS SDK"
             description="Découvrez comment configurer votre compte, réseau, utiliser le
         faucet, envoyer / simuler des transactions, et plus"
           />
         </CardGrid>

      4. Construisez votre app !

         La configuration développeur pour utiliser Aptos dans votre frontend est maintenant complète.
         Découvrez nos autres outils qui rationalisent le processus de développement

         <CardGrid>
           <LinkCard
             href="/build/indexer"
             title="Indexeur"
             description="Interrogez efficacement l'état on-chain comme les soldes,
         activité de transaction, données de token, et plus"
           />

           <LinkCard
             href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript"
             title="Exemples TS SDK"
             description="20+ exemples d'utilisation du TS
         SDK"
             target="_blank"
           />

           <LinkCard href="https://geomi.dev/" title="Geomi" description="Vous atteignez les limites de taux pour Fullnode API / Indexeurs ? Obtenez une clé API ici" target="_blank" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Smart Contract */}

  <TabItem label="Contrat Intelligent">
    <Steps>
      1. Installer CLI

         <CardGrid>
           <LinkCard href="/build/cli" title="Aptos CLI" description="Instructions pour installer Aptos CLI" />
         </CardGrid>

      2. Configurer l'Éditeur ou IDE

         Ajoutez les extensions suivantes à votre éditeur de choix pour faciliter le développement Move

         <Tabs>
           <TabItem label="IDE JetBrains">
             <CardGrid>
               <LinkCard
                 href="https://plugins.jetbrains.com/plugin/14721-move-language"
                 title="Move on Aptos"
                 description="Serveur de langage et surligneur de syntaxe pour
         les IDE JetBrains comme CLion, Rust Rover, WebStorm,
         IntelliJ"
                 target="_blank"
               />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=AptosLabs.move-on-aptos" title="Extension Move on Aptos" description="IDE Aptos Move complet pour VSCode" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="Cursor, Kiro, etc.">
             <CardGrid>
               <LinkCard href="https://open-vsx.org/extension/aptoslabs/move-on-aptos" title="Extension Move on Aptos" description="IDE Aptos Move complet pour les plateformes OpenVSX" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. Créer un Contrat Intelligent

         Naviguez vers votre dossier d'application et initialisez un nouveau contrat intelligent en faisant :

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. Construire, Compiler et Déployer le Contrat Intelligent !

         La configuration développeur pour utiliser Aptos pour les contrats intelligents est maintenant complète.
         Pour plus d'infos, voir le lien vers le tutoriel Dapp ci-dessous

         <CardGrid>
           <LinkCard
             href="/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file"
             title="Guide Création Contrat Intelligent"
             description="Un guide de liste de tâches facile pour configurer un contrat
         intelligent avec Move"
           />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* create-aptos-dapp */}

  <TabItem label="Créer une DApp Aptos">
    <Steps>
      1. Installer create-aptos-dapp

         Exécutez la commande ci-dessous pour installer une dApp à partir d'un modèle en quelques secondes :

         <Tabs>
           { /* npx */ }

           <TabItem label="npx">
             ```shellscript filename="Terminal"
             npx create-aptos-dapp@latest
             ```
           </TabItem>

           { /* pnpx */ }

           <TabItem label="pnpx">
             ```shellscript filename="Terminal"
             pnpx create-aptos-dapp@latest
             ```
           </TabItem>
         </Tabs>

      2. Suivez les invites

         Suivez les invites CLI pour sélectionner un nom, [modèle](/build/create-aptos-dapp#templates), et réseau pour votre nouvelle dApp.

         ![cad](~/images/cad-video.gif)

      3. Commencez à construire et personnaliser votre nouvelle dApp !

         Naviguez vers votre nouveau projet et ouvrez dans votre IDE préféré pour continuer à construire !

         Suivez le fichier `README.md` généré pour les prochaines étapes.

      4. Continuez à lire

         <CardGrid>
           <LinkCard href="/build/create-aptos-dapp" title="Créer une DApp Aptos" description="Obtenez plus d'informations sur l'outil" />

           <LinkCard href="/build/create-aptos-dapp#templates" title="Modèles" description="Parcourez les modèles prédéfinis" />

           <LinkCard href="/build/create-aptos-dapp/faq" title="FAQ" description="Obtenez de l'aide pour les problèmes et questions courants" />
         </CardGrid>
    </Steps>
  </TabItem>
</Tabs>
