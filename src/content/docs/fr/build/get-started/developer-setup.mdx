Voici un moyen facile de configurer votre environnement selon le type de développement.

<Tabs>
  {/* Frontend */}

  <TabItem label="Frontend">
    <Steps>
      1. Initialiser le Projet Frontend

         Voici quelques exemples de choix populaires :

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. Installer @aptos-labs/ts-sdk

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. Configurer le SDK TS

         <CardGrid>
           <LinkCard
             href="/fr/build/sdks/ts-sdk/quickstart"
             title="Démarrage Rapide SDK TS"
             description="Voyez comment configurer votre compte, réseau, utiliser le
         faucet, envoyer / simuler des transactions, et plus"
           />
         </CardGrid>

      4. Construisez votre application !

         <CardGrid>
           <LinkCard href="/fr/build/guides/build-e2e-dapp" title="Construire une DApp E2E" description="Apprenez à construire une application décentralisée complète de bout en bout sur Aptos" />

           <LinkCard href="/fr/build/guides/first-transaction" title="Votre Première Transaction" description="Apprenez à créer votre premier compte Aptos et à soumettre votre première transaction" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Contrats Intelligents / Move */}

  <TabItem label="Contrats Intelligents">
    <Steps>
      1. Installer la CLI Aptos

         <CardGrid>
           <LinkCard href="/fr/build/cli/install-cli" title="Installer la CLI Aptos" description="Installez la CLI Aptos sur Mac, Windows ou Linux" />
         </CardGrid>

      2. Créer un nouveau projet Move

         ```shellscript filename="Terminal"
         aptos move init --name mon-premier-package
         ```

      3. Écrire votre module Move

         <CardGrid>

         </CardGrid>

      4. Tester et déployer

           <LinkCard href="/fr/build/smart-contracts/book/unit-testing" title="Tests Unitaires" description="Apprenez à écrire et exécuter des tests unitaires pour vos modules Move" />

    </Steps>
  </TabItem>

  {/* Nœud Complet */}

  <TabItem label="Nœud Complet">
    <Steps>
      1. Configurer l'environnement

         Assurez-vous d'avoir les prérequis système :

         - **Processeur** : 8 cœurs, 16 threads (Intel Xeon Skylake ou plus récent)
         - **Mémoire** : 32 Go de RAM
         - **Stockage** : 2 To de disque NVMe
         - **Réseau** : 1 Gbit/s de bande passante

      2. Choisir votre méthode de déploiement

         <CardGrid>
           <LinkCard href="/fr/network/nodes/full-node/deployments/using-docker" title="Utiliser Docker" description="Déployez en utilisant Docker Compose (recommandé)" />

           <LinkCard href="/fr/network/nodes/full-node/deployments/using-aptos-core" title="Utiliser Aptos Core" description="Construisez et exécutez depuis le code source" />

           <LinkCard href="/fr/network/nodes/full-node/deployments/using-gcp" title="Utiliser GCP" description="Déployez sur Google Cloud Platform" />
         </CardGrid>

      3. Configurer votre nœud

         <CardGrid>
           <LinkCard href="/fr/network/nodes/full-node/modify" title="Configurer votre Nœud Complet" description="Apprenez à configurer et personnaliser votre nœud complet" />
         </CardGrid>

      4. Surveiller et maintenir

         <CardGrid>
           <LinkCard href="/fr/network/nodes/measure/node-health-checker" title="Vérificateur de Santé du Nœud" description="Surveillez la santé et les performances de votre nœud" />

           <LinkCard href="/fr/network/nodes/measure/important-metrics" title="Métriques Importantes" description="Métriques clés à surveiller pour votre nœud" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Nœud Validateur */}

  <TabItem label="Nœud Validateur">
    <Steps>
      1. Comprendre les prérequis

         <CardGrid>
           <LinkCard href="/fr/network/nodes/validator-node" title="Guide du Nœud Validateur" description="Aperçu complet de l'exploitation d'un nœud validateur" />
         </CardGrid>

      2. Configurer l'environnement

         Configuration minimale requise :

         - **Processeur** : 16 cœurs, 32 threads (Intel Xeon Skylake ou plus récent)
         - **Mémoire** : 64 Go de RAM
         - **Stockage** : 4 To de disque NVMe
         - **Réseau** : 1 Gbit/s de bande passante

      3. Déployer vos nœuds

           <LinkCard href="/fr/network/nodes/validator-node/deploy-nodes" title="Déployer les Nœuds" description="Déployez votre nœud validateur et votre nœud complet validateur" />

           <LinkCard href="/fr/network/nodes/validator-node/connect-nodes" title="Connecter au Réseau" description="Connectez vos nœuds au réseau de validateurs" />

    </Steps>
  </TabItem>
