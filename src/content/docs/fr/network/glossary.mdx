## A

### Accumulator Root Hash

- Un **accumulator root hash** est le hash racine d'un [accumulateur Merkle](https://eprint.iacr.org/2009/625.pdf).

### Account (Compte)

- Un **compte** dans la blockchain Aptos est un conteneur pour un nombre arbitraire de [modules Move](#move-module) et [ressources Move](#move-resources).
- L'état de chaque compte est composé de code et de données.
- Le compte est identifié par [l'adresse du compte](#account-address).

Voir [Comptes](/network/blockchain/accounts) pour plus d'informations.

### Account Address (Adresse du Compte)

- Une **adresse de compte** est l'adresse d'un compte Aptos.
- L'adresse de compte fait référence à une destination spécifique sur le réseau Aptos. L'adresse dicte la destination et la source d'un montant spécifique d'actifs échangés par deux parties sur la blockchain.
- Les adresses Aptos sont des chaînes hexadécimales de 64 caractères (32 octets). Souvent, ces chaînes sont préfixées par `0x` et pour les 16 premières adresses, les zéros en tête sont exclus (ex. `0x1`).

### API

- Une **Application Programming Interface (API)** est un ensemble de protocoles et d'outils qui permettent aux utilisateurs d'interagir avec les nœuds de blockchain Aptos et les réseaux clients via des applications externes. Aptos offre une API REST pour communiquer avec nos nœuds.
- Voir [documentation](/build/apis) pour plus de détails.

### APT

Le **jeton Aptos (APT)** est le jeton natif de la blockchain Aptos utilisé pour payer les frais de réseau et de transaction.

### Aptos

**Aptos** est une blockchain Layer 1 pour tout le monde. Elle utilise le langage de programmation Move et a lancé son mainnet le 17-10-2022 pour redéfinir l'expérience utilisateur web3. La blockchain Aptos est dédiée à créer de meilleures expériences utilisateur grâce à une vitesse accrue, sécurité, scalabilité, fiabilité et utilisabilité avec des coûts de transaction faibles. Le mot "Aptos" signifie "Les Gens" dans la langue Ohlone. En savoir plus sur la blockchain Aptos sur le [site officiel Aptos](https://aptosfoundation.org).

### AptosBFT

- **AptosBFT** est l'algorithme de consensus BFT du protocole Aptos.
- AptosBFT est basé sur Jolteon.

### Aptos Blockchain

- La **blockchain Aptos** est un registre de transactions immuables convenues par les validateurs sur le réseau Aptos (le réseau de validateurs).

### Aptos Name Service (ANS)

- Le **Aptos Name Service (ANS)** est un service d'adressage de noms décentralisé pour la blockchain Aptos. Un nom Aptos est un nom de domaine _.apt_ lisible par l'homme utilisé à la place d'une clé publique, par exemple _love.apt_.
- Ce service permet également aux utilisateurs d'enregistrer des noms de sous-domaine en plus du domaine enregistré. En savoir plus sur : [Aptosnames.com](https://www.aptosnames.com/)

### Aptos Framework

Le **Aptos Framework** définit l'API publique pour les mises à jour de blockchain et la structure des données on-chain. Il définit la logique métier et le contrôle d'accès pour les trois piliers clés de la fonctionnalité Aptos : paiements, trésorerie et gouvernance on-chain. Il est implémenté comme un ensemble de modules écrits dans le langage de programmation Move et stockés on-chain sous forme de bytecode Move.

### Aptos Node

Un **nœud Aptos** est une entité peer du réseau Aptos qui suit l'état de la blockchain Aptos. Il y a deux types de nœuds Aptos : [validateurs](#validator) et [nœuds complets](#fullnodes).

## B

### Blocks (Blocs)

- Sur Aptos, les blocs sont un lot de [transactions](#transaction) validées en même temps.
- Le numéro de bloc est analogue à la "hauteur de bloc" dans la littérature blockchain.

### Byzantine Fault Tolerance (BFT)

- **Byzantine Fault Tolerance (BFT)** est la propriété d'un système distribué de résister aux pannes byzantines (défaillances arbitraires).
- BFT garantit qu'un système continuera à fonctionner correctement même si certains nœuds échouent ou agissent de manière malveillante.

## C

### Consensus

- Le **consensus** est le mécanisme par lequel les nœuds d'un réseau distribué arrivent à un accord sur l'état de la blockchain.
- Aptos utilise AptosBFT comme algorithme de consensus.

## D

### Devnet

- **Devnet** est le réseau de développement Aptos où les développeurs peuvent tester leurs applications et contrats intelligents.
- Devnet est mis à jour régulièrement avec les dernières fonctionnalités mais peut être réinitialisé sans avertissement.

## E

### Events (Événements)

- Les **événements** sont des messages émis par les contrats intelligents Move pendant l'exécution.
- Les événements permettent aux applications off-chain de réagir aux changements d'état on-chain.

## F

### Fullnodes (Nœuds Complets)

- Un **nœud complet (fullnode)** est un nœud qui maintient une copie complète de l'état de la blockchain.
- Contrairement aux validateurs, les fullnodes ne participent pas au consensus mais peuvent valider les transactions.

## G

### Gas

- Le **gaz** est l'unité qui mesure le coût computationnel des transactions sur Aptos.
- Le gaz est payé en APT et couvre l'exécution, le stockage et les frais de réseau.

### Governance (Gouvernance)

- La **gouvernance** fait référence au processus par lequel les détenteurs de tokens peuvent proposer et voter sur les changements du protocole Aptos.
- La gouvernance on-chain permet aux détenteurs d'APT de participer directement aux décisions du réseau.

## M

### Mainnet

- **Mainnet** est le réseau principal de production d'Aptos où les transactions ont une valeur réelle.
- Mainnet est le réseau officiel et stable utilisé par les utilisateurs finaux et les applications de production.

### Merkle Trees

- Les **arbres Merkle** sont des structures de données utilisées pour vérifier efficacement l'intégrité des données dans les blockchains.
- Ils permettent de vérifier qu'un élément fait partie d'un ensemble plus grand sans révéler l'ensemble complet.

### Move

- **Move** est le langage de programmation de contrats intelligents conçu spécifiquement pour Aptos.
- Move met l'accent sur la sécurité, la modularité et la vérification formelle.

### Move Module

- Un **module Move** est une unité de code déployée sur la blockchain Aptos.
- Les modules contiennent des définitions de types, des fonctions et des ressources.

### Move Resources

- Les **ressources Move** sont des instances de structures qui stockent des données sur la blockchain.
- Contrairement aux modules, les ressources contiennent des données et non du code.

## R

### Resources (Ressources)

- Voir [Move Resources](#move-resources).

## S

### Staking

- Le **staking** est le processus de verrouillage des tokens APT pour soutenir un validateur et gagner des récompenses.
- Les détenteurs d'APT peuvent miser leurs tokens pour aider à sécuriser le réseau et gagner des récompenses.

## T

### Testnet

- **Testnet** est le réseau de test Aptos où les développeurs peuvent expérimenter sans risque financier.
- Testnet reflète généralement les fonctionnalités de mainnet mais peut être réinitialisé.

### Transaction

- Une **transaction** est une opération soumise à la blockchain Aptos pour changer son état.
- Les transactions peuvent inclure des transferts de tokens, des déploiements de contrats, etc.

## V

### Validator (Validateur)

- Un **validateur** est un nœud qui participe au consensus et aide à maintenir la sécurité du réseau.
- Les validateurs doivent miser des tokens APT et suivre les règles du protocole pour gagner des récompenses.
