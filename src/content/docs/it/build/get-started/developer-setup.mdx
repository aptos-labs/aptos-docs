import { CardGrid, LinkCard, Steps, TabItem, Tabs } from '@astrojs/starlight/components';

Ecco un modo semplice per configurare il tuo ambiente a seconda del tipo di sviluppo.

<Tabs>
  {/* Frontend */}

  <TabItem label="Frontend">
    <Steps>
      1. Inizializza Progetto Frontend

         Ecco alcuni esempi di scelte popolari:

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. Installa @aptos-labs/ts-sdk

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. Configura TS SDK

         <CardGrid>
           <LinkCard href="/it/build/sdks/ts-sdk/quickstart" title="Guida Rapida TS SDK" description="Scopri come configurare il tuo account, la rete, usare il faucet, inviare / simulare transazioni e altro" />
         </CardGrid>

      4. Costruisci la tua app!

         La configurazione per sviluppatori per usare Aptos nel tuo frontend è ora completa.
         Scopri i nostri altri strumenti che semplificano il processo di sviluppo

         <CardGrid>
           <LinkCard href="/it/build/indexer" title="Indexer" description="Interroga efficientemente lo stato on-chain come saldi, attività delle transazioni, dati dei token e altro" />

           <LinkCard href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript" title="Esempi TS SDK" description="20+ Esempi di come usare l'TS SDK" target="_blank" />

           <LinkCard href="https://build.aptoslabs.com/" title="Aptos Build" description="Hai raggiunto i limiti di velocità per le API Fullnode / Indexer? Ottieni una Chiave API qui" target="_blank" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Smart Contract */}

  <TabItem label="Smart Contract">
    <Steps>
      1. Installa CLI

         <CardGrid>
           <LinkCard href="/it/build/cli" title="CLI di Aptos" description="Istruzioni su come installare la CLI di Aptos" />
         </CardGrid>

      2. Configura Editor o IDE

         Aggiungi le seguenti estensioni al tuo editor di scelta per rendere lo sviluppo Move più facile

         <Tabs>
           <TabItem label="IDE JetBrains">
             <CardGrid>
               <LinkCard href="https://plugins.jetbrains.com/plugin/14721-move-language" title="Move on Aptos" description="Server di linguaggio e evidenziatore di sintassi per IDE JetBrains come CLion, Rust Rover, WebStorm, IntelliJ" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=AptosLabs.move-on-aptos" title="Estensione Move on Aptos" description="IDE Move di Aptos completo per VSCode" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="Cursor, Kiro, ecc.">
             <CardGrid>
               <LinkCard href="https://open-vsx.org/extension/aptoslabs/move-on-aptos" title="Estensione Move on Aptos" description="IDE Move di Aptos completo per piattaforme basate su OpenVSX" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. Crea Smart Contract

         Naviga nella cartella della tua applicazione e inizializza un nuovo smart contract facendo:

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. Costruisci, Compila e Distribuisci Smart Contract!

         La configurazione per sviluppatori per usare Aptos per smart contract è ora completa.
         Per maggiori informazioni vedi il link al tutorial Dapp sotto

         <CardGrid>
           <LinkCard href="/it/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file" title="Guida Crea Smart Contract" description="Una semplice guida todo list su come configurare uno smart contract con Move" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* create-aptos-dapp */}

  <TabItem label="Crea Aptos Dapp">
    <Steps>
      1. Installa create-aptos-dapp

         Esegui il comando sotto per installare una dApp da un template in secondi:

         <Tabs>
           { /* npx */ }

           <TabItem label="npx">
             ```shellscript filename="Terminal"
             npx create-aptos-dapp@latest
             ```
           </TabItem>

           { /* pnpx */ }

           <TabItem label="pnpx">
             ```shellscript filename="Terminal"
             pnpx create-aptos-dapp@latest
             ```
           </TabItem>
         </Tabs>

      2. Segui i prompt

         Segui i prompt della CLI per selezionare un nome, [template](/it/build/create-aptos-dapp#templates) e rete per la tua nuova dApp.

         ![cad](~/images/cad-video.gif)

      3. Inizia a costruire e personalizzare la tua nuova dApp!

         Naviga nel tuo nuovo progetto e aprilo nel tuo IDE preferito per continuare a costruire!

         Segui il file `README.md` generato per i prossimi passi.

      4. Continua a leggere

         <CardGrid>
           <LinkCard href="/it/build/create-aptos-dapp" title="Crea Aptos Dapp" description="Ottieni maggiori informazioni sullo strumento" />

           <LinkCard href="/it/build/create-aptos-dapp#templates" title="Template" description="Esplora template predisposti" />

           <LinkCard href="/it/build/create-aptos-dapp/faq" title="FAQ" description="Ottieni aiuto per problemi comuni e domande" />
         </CardGrid>
    </Steps>
  </TabItem>
</Tabs>
