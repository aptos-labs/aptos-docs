Aptos CLIの特定のバージョンが必要な場合は,Aptosのソースコードから直接ビルドすることができます.このインストール方法は,主にTestnet/Mainnetにまだ反映されていないDevnetの特定の機能と対話する場合に使用されます.また,既存のリリース/プリコンパイル済みバイナリがうまく動作しないアーキテクチャを使用している場合にも,これらの手順に従うことをお勧めします.

この上級者向けの方法が必要ない場合は,[こちら](/ja/build/cli)で通常のインストール手順を確認できます.

## macOS / Linuxでのインストール



<Steps>

1. の手順に従います.

2. の手順に従ってcargoがインストールされていることを確認します.

3. CLIツールをビルドします:cargo build --package aptos --profile cli

   バイナリは`target/cli/aptos`で利用可能になります.

4. (オプション)この実行ファイルをPATHの通った場所に移動します.

5. target/cli/aptos helpを実行してインストールが成功したことを確認します.

   これらのヘルプ指示は,特定のコマンドの詳細なガイドとしても役立ちます.

</Steps>



## Windowsでのインストール



<Steps>

1. の手順に従ってソースからAptosをビルドします.

2. の手順に従ってcargoがインストールされていることを確認します.

3. CLIツールをビルドします:cargo build --package aptos --profile cli

   バイナリは`target\cli\aptos.exe`で利用可能になります.

4. (オプション)この実行ファイルをPATHの通った場所に移動します.

5. target\cli\aptos.exe helpを実行してインストールが成功したことを確認します.

   これらのヘルプ指示は,特定のコマンドの詳細なガイドとしても役立ちます.

</Steps>
