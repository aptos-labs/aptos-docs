title: "開発者セットアップ"

開発タイプに応じた環境構築の簡単な方法をご紹介します.

<Tabs>
  {/* フロントエンド */}

  <TabItem label="フロントエンド">
    <Steps>
      1. フロントエンドプロジェクトの初期化

         一般的な選択肢の例をご紹介します:

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. @aptos-labs/ts-sdkのインストール

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. TS SDKのセットアップ

         <CardGrid>
           <LinkCard
             href="/ja/build/sdks/ts-sdk/quickstart"
             title="TS SDK クイックスタート"
             description="アカウントの設定、ネットワーク、フォーセットの使用方法、
         トランザクションの送信/シミュレーションなどについて学びましょう"
           />
         </CardGrid>

      4. アプリを構築しましょう!

         Aptosをフロントエンドで使用するための開発環境のセットアップが完了しました.
         開発プロセスを効率化する他のツールもご確認ください

         <CardGrid>
           <LinkCard
             href="/ja/build/indexer"
             title="インデクサー"
             description="残高、トランザクション履歴、トークンデータなど、
         オンチェーン状態を効率的にクエリできます"
           />

           <LinkCard href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript" title="TS SDKの使用例" description="TS SDKの使用方法を示す20以上の例" target="_blank" />

           <LinkCard
             href="https://developers.aptoslabs.com/"
             title="APIゲートウェイ"
             description="フルノードAPI/インデクサーのレート制限に達しましたか？
         ここでAPIキーを取得できます"
             target="_blank"
           />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* スマートコントラクト */}

  <TabItem label="スマートコントラクト">
    <Steps>
      1. CLIのインストール

         <CardGrid>
           <LinkCard href="/ja/build/cli" title="Aptos CLI" description="Aptos CLIのインストール方法について" />
         </CardGrid>

      2. エディタまたはIDEのセットアップ

         Move開発を容易にするために,お好みのエディタに以下の拡張機能を追加してください

         <Tabs>
           <TabItem label="JetBrains IDE">
             <CardGrid>
               <LinkCard
                 href="https://plugins.jetbrains.com/plugin/14721-move-language"
                 title="Move Language"
                 description="CLion、Rust Rover、WebStorm、
         IntelliJなどのJetBrains IDE用の言語サーバーと構文ハイライター"
                 target="_blank"
               />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=MoveBit.aptos-move-analyzer" title="Aptos Move Analyzer" description="VSCode用の言語サーバーと構文ハイライター" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. スマートコントラクトの作成

         アプリケーションフォルダに移動し,以下のコマンドで新しいスマートコントラクトを初期化します:

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. スマートコントラクトのビルド,コンパイル,デプロイ!

         Aptosでスマートコントラクトを使用するための開発環境のセットアップが完了しました.
         詳細については,以下のDappチュートリアルのリンクをご覧ください

         <CardGrid>
           <LinkCard
             href="/ja/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file"
             title="スマートコントラクト作成ガイド"
             description="Moveでスマートコントラクトをセットアップする方法を
         簡単なTodoリストで学ぶガイド"
           />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* create-aptos-dapp */}

  <TabItem label="Create Aptos Dapp">
    <Steps>
      1. create-aptos-dappのインストール

         以下のコマンドを実行して,テンプレートからdAppを数秒でインストールできます:

         <Tabs>
           { /* npx */ }

           <TabItem label="npx">
             ```shellscript filename="Terminal"
             npx create-aptos-dapp@latest
             ```
           </TabItem>

           { /* pnpx */ }

           <TabItem label="pnpx">
             ```shellscript filename="Terminal"
             pnpx create-aptos-dapp@latest
             ```
           </TabItem>
         </Tabs>

      2. プロンプトに従って設定

         CLIプロンプトに従って,新しいdAppの名前,[テンプレート](/ja/build/create-aptos-dapp#templates),ネットワークを選択してください.

         ![cad](/images/cad-video.gif)

      3. 新しいdAppの構築とカスタマイズを始めましょう!

         新しいプロジェクトに移動し,お好みのIDEで開いて開発を続けましょう!

         次のステップについては,生成された`README.md`ファイルをご確認ください.

      4. 関連情報

         <CardGrid>
           <LinkCard href="/ja/build/create-aptos-dapp" title="Create Aptos Dapp" description="ツールの詳細情報" />

           <LinkCard href="/ja/build/create-aptos-dapp#templates" title="テンプレート" description="用意されているテンプレートを閲覧" />

           <LinkCard href="/ja/build/create-aptos-dapp/faq" title="FAQ" description="よくある問題や質問への回答" />
         </CardGrid>
    </Steps>
  </TabItem>
</Tabs>
