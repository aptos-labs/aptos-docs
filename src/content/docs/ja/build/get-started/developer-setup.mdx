title: "開発者セットアップ"

import { CardGrid, LinkCard, Steps, TabItem, Tabs } from '@astrojs/starlight/components';

開発の種類に応じて環境をセットアップする簡単な方法をここに示します。

<Tabs>
  {/* Frontend */}

  <TabItem label="フロントエンド">
    <Steps>
      1. フロントエンドプロジェクトを初期化

         人気の選択肢の例をいくつか示します：

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. @aptos-labs/ts-sdkをインストール

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. TS SDKをセットアップ

         <CardGrid>
           <LinkCard href="/ja/build/sdks/ts-sdk/quickstart" title="TS SDK クイックスタート" description="アカウントのセットアップ、ネットワーク、フォーセットの使用、トランザクションの送信/シミュレーションなどの方法を確認" />
         </CardGrid>

      4. アプリを構築！

         フロントエンドでAptosを使用するための開発者セットアップが完了しました。
         開発プロセスを効率化する他のツールもチェックしてください

         <CardGrid>
           <LinkCard href="/ja/build/indexer" title="インデクサー" description="残高、トランザクション活動、トークンデータなどのオンチェーン状態を効率的にクエリ" />

           <LinkCard href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript" title="TS SDK の例" description="TS SDKの使用方法についての20以上の例" target="_blank" />

           <LinkCard href="https://build.aptoslabs.com/" title="Aptos Build" description="フルノードAPI/インデクサーのレート制限に達していますか？ここでAPIキーを取得" target="_blank" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Smart Contract */}

  <TabItem label="スマートコントラクト">
    <Steps>
      1. CLIをインストール

         <CardGrid>
           <LinkCard href="/ja/build/cli" title="Aptos CLI" description="Aptos CLIのインストール方法" />
         </CardGrid>

      2. エディターまたはIDEをセットアップ

         Move開発を簡単にするために、選択したエディターに以下の拡張機能を追加してください

         <Tabs>
           <TabItem label="JetBrains IDE">
             <CardGrid>
               <LinkCard href="https://plugins.jetbrains.com/plugin/14721-move-language" title="Move on Aptos" description="CLion、Rust Rover、WebStorm、IntelliJなどのJetBrains IDE用の言語サーバーとシンタックスハイライター" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=AptosLabs.move-on-aptos" title="Move on Aptos 拡張機能" description="VSCode用のフル機能Aptos Move IDE" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="Cursor、Kiroなど">
             <CardGrid>
               <LinkCard href="https://open-vsx.org/extension/aptoslabs/move-on-aptos" title="Move on Aptos 拡張機能" description="OpenVSX対応プラットフォーム用のフル機能Aptos Move IDE" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. スマートコントラクトを作成

         アプリケーションフォルダに移動し、以下のコマンドで新しいスマートコントラクトを初期化します：

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. スマートコントラクトをビルド、コンパイル、デプロイ！

         スマートコントラクト用のAptos開発者セットアップが完了しました。
         詳細については、以下のDappチュートリアルへのリンクを参照してください

         <CardGrid>
           <LinkCard href="/ja/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file" title="スマートコントラクト作成ガイド" description="Moveでスマートコントラクトをセットアップする方法の簡単なTodoリストガイド" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* create-aptos-dapp */}

  <TabItem label="Create Aptos Dapp">
    <Steps>
      1. create-aptos-dappをインストール

         テンプレートから数秒でdAppをインストールするには、以下のコマンドを実行します：

         <Tabs>
           { /* npx */ }

           <TabItem label="npx">
             ```shellscript filename="Terminal"
             npx create-aptos-dapp@latest
             ```
           </TabItem>

           { /* pnpx */ }

           <TabItem label="pnpx">
             ```shellscript filename="Terminal"
             pnpx create-aptos-dapp@latest
             ```
           </TabItem>
         </Tabs>

      2. プロンプトに従う

         CLIプロンプトに従って、新しいdAppの名前、[テンプレート](/ja/build/create-aptos-dapp#templates)、ネットワークを選択します。

         ![cad](~/images/cad-video.gif)

      3. 新しいdAppの構築とカスタマイズを開始！

         新しいプロジェクトに移動し、お気に入りのIDEで開いて構築を続けましょう！

         次の手順については、生成された`README.md`ファイルに従ってください。

      4. 続きを読む

         <CardGrid>
           <LinkCard href="/ja/build/create-aptos-dapp" title="Create Aptos Dapp" description="ツールについての詳細情報を取得" />

           <LinkCard href="/ja/build/create-aptos-dapp#templates" title="テンプレート" description="事前作成されたテンプレートを参照" />

           <LinkCard href="/ja/build/create-aptos-dapp/faq" title="FAQ" description="よくある問題と質問のヘルプを取得" />
         </CardGrid>
    </Steps>
  </TabItem>
</Tabs>
