import { CardGrid, LinkCard, Steps, TabItem, Tabs } from '@astrojs/starlight/components';

開発タイプに応じて環境を設定する簡単な方法です。

<Tabs>
  {/* Frontend */}

  <TabItem label="フロントエンド">
    <Steps>
      1. フロントエンドプロジェクトの初期化

         人気のある選択肢の例です：

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. @aptos-labs/ts-sdkのインストール

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. TS SDKの設定

         <CardGrid>
           <LinkCard
             href="/build/sdks/ts-sdk/quickstart"
             title="TS SDKクイックスタート"
             description="アカウント、ネットワークの設定方法、faucetの使用、
         トランザクションの送信/シミュレーションなどを確認してください"
           />
         </CardGrid>

      4. アプリを構築しましょう！

         フロントエンドでAptosを使用するための開発者設定が完了しました。
         開発プロセスを合理化する他のツールを確認してください

         <CardGrid>
           <LinkCard
             href="/build/indexer"
             title="Indexer"
             description="残高、トランザクション活動、トークンデータなどの
         オンチェーン状態を効率的にクエリ"
           />

           <LinkCard
             href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript"
             title="TS SDKの例"
             description="TS SDKの使用方法に関する
         20以上の例"
             target="_blank"
           />

           <LinkCard href="https://geomi.dev/" title="Geomi" description="Fullnode API / Indexerのレート制限に達しましたか？ ここでAPIキーを取得してください" target="_blank" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Smart Contract */}

  <TabItem label="スマートコントラクト">
    <Steps>
      1. CLIのインストール

         <CardGrid>
           <LinkCard href="/build/cli" title="Aptos CLI" description="Aptos CLIのインストール手順" />
         </CardGrid>

      2. エディタまたはIDEの設定

         選択したエディタに以下の拡張機能を追加してMove開発を簡単にします

         <Tabs>
           <TabItem label="JetBrains IDE">
             <CardGrid>
               <LinkCard
                 href="https://plugins.jetbrains.com/plugin/14721-move-language"
                 title="Move on Aptos"
                 description="CLion、Rust Rover、WebStormなどのための
         言語サーバーと構文ハイライター、
         IntelliJ"
                 target="_blank"
               />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=AptosLabs.move-on-aptos" title="Move on Aptos拡張" description="VSCodeのための完全なAptos Move IDE" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="Cursor、Kiroなど">
             <CardGrid>
               <LinkCard href="https://open-vsx.org/extension/aptoslabs/move-on-aptos" title="Move on Aptos拡張" description="OpenVSXベースのプラットフォームのための完全なAptos Move IDE" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. スマートコントラクトの作成

         アプリケーションのフォルダに移動して、次の操作で新しいスマートコントラクトを初期化します：

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. スマートコントラクトの構築、コンパイル、展開！

         スマートコントラクトにAptosを使用するための開発者設定が完了しました。
         詳細については以下のDappチュートリアルリンクを参照してください

         <CardGrid>
           <LinkCard
             href="/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file"
             title="スマートコントラクト作成ガイド"
             description="Moveでスマートコントラクトを設定するための
         簡単なTodoリストガイド"
           />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* create-aptos-dapp */}

  <TabItem label="Aptos DApp作成">
    <Steps>
      1. create-aptos-dappのインストール

         数秒でテンプレートからdAppをインストールするには、以下のコマンドを実行してください：

         <Tabs>
           { /* npx */ }

           <TabItem label="npx">
             ```shellscript filename="Terminal"
             npx create-aptos-dapp@latest
             ```
           </TabItem>

           { /* pnpx */ }

           <TabItem label="pnpx">
             ```shellscript filename="Terminal"
             pnpx create-aptos-dapp@latest
             ```
           </TabItem>
         </Tabs>

      2. プロンプトに従う

         新しいdAppの名前、[テンプレート](/build/create-aptos-dapp#templates)、ネットワークを選択するにはCLIプロンプトに従ってください。

         ![cad](~/images/cad-video.gif)

      3. 新しいdAppの構築とカスタマイズを開始しましょう！

         新しいプロジェクトに移動して、構築を続けるために好きなIDEで開いてください！

         次のステップについては生成された`README.md`ファイルに従ってください。

      4. 読み続けてください

         <CardGrid>
           <LinkCard href="/build/create-aptos-dapp" title="Aptos DApp作成" description="ツールについての詳細情報" />

           <LinkCard href="/build/create-aptos-dapp#templates" title="テンプレート" description="プレメイドテンプレートを閲覧" />

           <LinkCard href="/build/create-aptos-dapp/faq" title="FAQ" description="一般的な問題と質問に対するヘルプ" />
         </CardGrid>
    </Steps>
  </TabItem>
</Tabs>
