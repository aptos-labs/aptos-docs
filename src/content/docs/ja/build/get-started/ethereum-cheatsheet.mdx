### 高レベルの概要

| 機能                        | Ethereum                                                                                                                       | Aptos                                                                 |
|-----------------------------|-------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------|
| **スマートコントラクト**    | Solidity, EVM                                                                                                                  | Move, MoveVM                                                          |
| **利点**                    | 成熟、大規模採用                                                                                                              | スケーラビリティ、低遅延、予測可能な手数料                           |
| **トランザクション手数料**  | 変動的、高くなる可能性                                                                                                        | より低く、より予測可能                                               |
| **アカウントアドレス**      | 160ビット                                                                                                                      | 256ビット                                                             |
| **アカウント構造**          | 単一フィールドの残高、nonce使用                                                                                                | モジュールとリソース、シーケンス番号使用                             |
| **データストレージ**        | Patricia Merkleツリー                                                                                                          | リソースとモジュールによるグローバルストレージ                      |
| **ストレージマインドセット** | コントラクトベースストレージ                                                                                                   | コードとデータのためのアカウント中心マインドセット                   |
| **サンプルコード**          | [ERC-20](https://github.com/OpenZeppelin/openzeppelin-contracts/tree/master/contracts/token/ERC20)                             | [Fungible Asset](https://github.com/aptos-labs/aptos-core/blob/main/aptos-move/framework/aptos-framework/sources/fungible_asset.move) |
| **呼び出し元ID**           | `msg.sender`                                                                                                                   | `&signer`参照                                                         |
| **アップグレード可能性**    | プロキシパターン                                                                                                               | 直接モジュールアップグレード                                          |
| **安全性とセキュリティ**    | リエントランスなどの攻撃に対して脆弱                                                                                            | 一般的な脆弱性を軽減                                                  |
| **ディスパッチタイプ**      | 動的ディスパッチ                                                                                                                | 静的ディスパッチ                                                       |
| **FT標準**                  | [ERC-20](https://docs.openzeppelin.com/contracts/4.x/erc20)                                                                    | [Coin](/build/smart-contracts/aptos-coin) (legacy) と [Fungible Asset](/build/smart-contracts/fungible-asset) |
| **NFT標準**                 | [ERC-721](https://docs.openzeppelin.com/contracts/4.x/erc721), [ERC-1155](https://docs.openzeppelin.com/contracts/4.x/erc1155) | [Digital Asset](/build/smart-contracts/digital-asset)                 |
| **ブロックチェーン相互作用** | [Ethers.jsライブラリ](https://docs.ethers.org/v6/)                                                                              | [Aptos TypeScript SDK](/build/sdks/ts-sdk)                            |

### トークン標準の詳細比較

|                        | Solidity                                                               | Move (Aptos)                                                                 |
|------------------------|------------------------------------------------------------------------|------------------------------------------------------------------------------|
| **トークン構造**      | 各トークンは独自のコントラクトです。                                  | すべてのトークンは単一の再利用可能なコントラクトを使用して型付けされた`Coin`または`FungibleAsset`です。 |
| **トークン標準**      | ERC20などの標準に準拠する必要があります；実装が異なる場合があります。 | すべてのトークンに対する統一されたインターフェースと実装。                  |
| **残高ストレージ**    | マッピング構造を使用してコントラクトに残高を保存。                    | **リソース指向残高**: 残高はユーザーのアカウントにリソースとして保存されます。リソースは任意に作成できないため、トークン価値の完全性を確保します。 |
| **転送メカニズム**    | 受信者の明示的な許可なしにトークンを転送できます。                    | 特定のケース(AptosCoinなど)を除き、トークンは通常転送のために受信者の`signer`権限が必要です。 |

<br />

### EVMとMove VMの詳細比較

- **EVM**: 柔軟性と動的ディスパッチで知られており、これは幅広いスマートコントラクト動作を可能にします。この柔軟性はしかし、並列実行とネットワーク操作における複雑性を引き起こす可能性があります。
- **Move VM**: VMとプログラミング言語間のより統合されたアプローチでセキュリティと効率性に焦点を当てています。そのデータストレージモデルはより優れた並列化を可能にし、静的ディスパッチ方法はセキュリティと予測可能性を向上させます。

<br />

|                                 | EVM (Ethereum Virtual Machine)                                         | Move VM (Move Virtual Machine)                                     |
|---------------------------------|------------------------------------------------------------------------|-------------------------------------------------------------------|
| **データストレージ**            | データはスマートコントラクトのストレージスペースに保存されます。      | データはスマートコントラクト、ユーザーアカウント、オブジェクトにまたがって保存されます。 |
| **並列化**                      | 共有ストレージスペースのため並列実行が制限されます。                  | 柔軟な分割ストレージ設計により、より多くの並列実行が可能になります。 |
| **VMと言語の統合**              | EVMとスマートコントラクト言語(例: Solidity)の別々のレイヤー。         | VMレイヤーとMove言語間のシームレスな統合、Rustで記述されたネイティブ関数がMoveで実行可能。 |
| **重要なネットワーク操作**      | ネットワーク操作の実装が複雑であまり直接的でない場合があります。      | バリデータセット管理などの重要な操作がMoveでネイティブに実装され、直接実行を可能にします。 |
| **関数呼び出し**                | 動的ディスパッチが任意のスマートコントラクト呼び出しを許可します。     | 静的ディスパッチがセキュリティと予測可能な動作に焦点を当てています。 |
| **型安全性**                    | コントラクト型が型の安全性のレベルを提供します。                     | Moveのモジュール構造とジェネリックが堅牢な型安全性を提供します。     |
| **トランザクション安全性**       | トランザクションの順序と安全性のためのnonceを使用します。             | トランザクションの順序と安全性のためのシーケンス番号を使用します。   |
| **認証ストレージ**              | スマートコントラクトストレージで可能。                               | Moveのリソースモデルを活用して可能。                              |
| **オブジェクトアクセシビリティ** | オブジェクトはグローバルにアクセスできません；スマートコントラクトスコープにバインドされます。 | オブジェクトのグローバルアクセシビリティが保証されます。            |
