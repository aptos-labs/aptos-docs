インデクサーAPIを使用すると,GraphQLクエリを使用してオンチェーン上のトークン,アカウント,トランザクション,イベントに関する豊富なデータにアクセスできます.**[こちら](/ja/build/indexer/aptos-hosted)からアクセスできます.**

一般的なクエリについては,サイドバーの例を参照してください.独自のクエリを作成する際は,このリファレンスガイドを参考に,関連するテーブルと適切なクエリの形式を決定してください.

<Aside type="caution">
  本番サービスでテーブルを使用する前に,このページの下部でそのテーブルが非推奨になっていないか確認してください.非推奨の場合は,非推奨でないテーブルに移行するためのガイダンスを参照してください.
</Aside>

<Aside type="note">
  `_by_pk`サフィックスのついたテーブルを検索する場合は,サフィックスなしのテーブル名で検索してください.`_by_pk`テーブルは,主キーによるクエリを容易にするために自動生成されています.
</Aside>

# インデクサーテーブルリファレンス

<Aside type="note">
  興味のあるテーブルを見つけるには,Ctrl + Fを使用してください!不明な点がある場合は,[インデクサーAPIアクセス](/ja/build/indexer/aptos-hosted)ページにリンクされているHasuraテーブルをクエリして,データの例を確認することもできます.
</Aside>

## 生データ

### `user_transactions`

ユーザートランザクション(システムではない)にフィルタリングされたトランザクション.

### `events`

トランザクションによって発行されたイベント.

### `block_metadata_transactions`

ブロックごとに1回発行されるシステムトランザクションのタイプで,タイムスタンプやエポックへのマッピングに便利です.
