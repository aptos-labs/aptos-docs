## A

### Accumulator Root Hash

- **accumulator root hash**は[Merkle accumulator](https://eprint.iacr.org/2009/625.pdf)のルートハッシュです。

### Account (アカウント)

- Aptosブロックチェーンの**アカウント**は、任意の数の[Moveモジュール](#move-module)と[Moveリソース](#move-resources)のためのコンテナです。
- 各アカウントの状態はコードとデータで構成されます。
- アカウントは[アカウントアドレス](#account-address)で識別されます。

[アカウント](/network/blockchain/accounts)の詳細についてはこちらをご覧ください。

### Account Address (アカウントアドレス)

- **アカウントアドレス**はAptosアカウントのアドレスです。
- アカウントアドレスはAptosネットワーク内の特定の宛先を指します。アドレスはブロックチェーン上で2者間で交換される特定の資産額の宛先と送信元を決定します。
- Aptosアドレスは64文字の16進文字列(32バイト)です。これらの文字列はしばしば`0x`で始まり、最初の16アドレスの先頭の0は省略されます(例: `0x1`)。

[アカウント](/network/blockchain/accounts)の詳細についてはこちらをご覧ください。

### API

- \*\*API(アプリケーション・プログラミング・インターフェース)\*\*は、外部アプリケーションを介してAptosブロックチェーンノードおよびクライアントネットワークと相互作用するためのプロトコルとツールのセットです。Aptosはノードとの通信のためのREST APIを提供します。
- 詳細は[ドキュメント](/build/apis)を参照してください。

### APT

\*\*Aptosトークン(APT)\*\*はネットワークおよびトランザクションフィーを支払うために使用されるAptosブロックチェーンのネイティブトークンです。

### Aptos

**Aptos**は全員のためのLayer 1ブロックチェーンです。Moveプログラミング言語を使用し、2022-10-17にメインネットをローンチしてweb3ユーザーエクスペリエンスを再定義しました。Aptosブロックチェーンは、増加した速度、セキュリティ、スケーラビリティ、信頼性、および低いトランザクションコストを通じて、より優れたユーザーエクスペリエンスを作成することに特化しています。"Aptos"という言葉はOhlone言語で"The People"を意味します。[公式Aptosウェブサイト](https://aptosfoundation.org)でAptosブロックチェーンについて詳しく学びましょう。

### AptosBFT

- **AptosBFT**はAptosプロトコルのBFTコンセンサスアルゴリズムです。
- AptosBFTはJolteonをベースとしています。

### Aptos Blockchain

- **Aptosブロックチェーン**はAptosネットワークのバリデーターによって合意された不変トランザクションの台帳です(バリデーターのネットワーク)。

### Aptos Name Service (ANS)

- \*\*Aptos Name Service(ANS)\*\*はAptosブロックチェーンのための分散型ネーミングアドレスサービスです。Aptos名は人間が読める\_.apt\_ドメイン名で、公開キーの代わりに使用されます(例: _love.apt_)。
- このサービスにより、ユーザーは登録ドメインに加えてサブドメイン名を登録することもできます。詳細は[Aptosnames.com](https://www.aptosnames.com)を参照してください。

### Aptos Framework

**Aptos Framework**はブロックチェーン更新のためのパブリックAPIとオンチェーンデータ構造を定義します。支払い、財務、およびオンチェーンガバナンスの3つのコアAptos機能の柱に対するビジネスロジックとアクセス制御を定義します。Moveプログラミング言語で書かれたモジュールのセットとして実装され、オンチェーンでMoveバイトコードとして保存されます。

### Aptos Node

**Aptosノード**はAptosブロックチェーンの状態を追跡するAptosネットワークのピアエンティティです。[バリデーター](#validator)と[全ノード](#fullnodes)の2種類のAptosノードがあります。

## B

### Blocks (ブロック)

- Aptosでは、ブロックは同時にコミットされる[トランザクション](#transaction)のバッチです。
- ブロック番号はブロックチェーン文献の"ブロック高"と類似しています。

### Byzantine Fault Tolerance (BFT)

- \*\*Byzantine Fault Tolerance(BFT)\*\*は分散システムがビザンチン障害(任意の障害)に耐性を持つ性質です。
- BFTは一部のノードが失敗したり悪意を持って動作したりしてもシステムが正しく動作し続けることを保証します。

## C

### Consensus

- **コンセンサス**は分散ネットワークのノードがブロックチェーンの状態について合意するメカニズムです。
- AptosはコンセンサスアルゴリズムとしてAptosBFTを使用します。

## D

### Devnet

- **Devnet**は開発者がアプリケーションとスマートコントラクトをテストできるAptos開発ネットワークです。
- Devnetは定期的に最新機能で更新されますが、警告なしにリセットされる場合があります。

## E

### Events (イベント)

- **イベント**はMoveスマートコントラクトが実行中に放出するメッセージです。
- イベントはオフチェーンアプリケーションがオンチェーン状態変化に反応することを可能にします。

## F

### Fullnodes (全ノード)

- \*\*全ノード(fullnode)\*\*はブロックチェーンの完全な状態のコピーを維持するノードです。
- バリデーターとは異なり、全ノードはコンセンサスに参加しませんが、トランザクションを検証できます。

## G

### Gas

- **ガス**はAptosでのトランザクションの計算コストを測定する単位です。
- ガスはAPTで支払われ、実行、ストレージ、およびネットワークフィーをカバーします。

### Governance (ガバナンス)

- **ガバナンス**とはAPTトークン保有者がAptosプロトコルの変更を提案・投票できるプロセスを指します。
- オンチェーンガバナンスによりAPT保有者はネットワーク決定に直接参加できます。

## M

### Mainnet

- **Mainnet**は実際のトランザクション価値を持つAptosのプライマリプロダクションネットワークです。
- Mainnetはエンドユーザーとプロダクションアプリケーションで使用される公式で安定したネットワークです。

### Merkle Trees

- **Merkleツリー**はブロックチェーンでのデータ整合性を効率的に検証するために使用されるデータ構造です。
- これにより、より大きなセットの一部であることを、より大きなセット全体を明らかにすることなく要素が確認できます。

### Move

- **Move**はAptos専用に設計されたスマートコントラクトプログラミング言語です。
- Moveはセキュリティ、モジュール性、および形式検証を重視します。

### Move Module

- **Moveモジュール**はAptosブロックチェーンに展開されたコードの単位です。
- モジュールにはタイプ定義、関数、およびリソースが含まれます。

### Move Resources

- **Moveリソース**はブロックチェーンにデータを保存する構造のインスタンスです。
- モジュールとは異なり、リソースにはデータが含まれ、コードは含まれません。

## S

### Staking

- **ステーキング**はバリデーターをサポートし報酬を受け取るためにAPTトークンをロックするプロセスです。
- APT保有者はネットワークのセキュリティを助け、報酬を受け取るためにトークンをステーキングできます。

## T

### Testnet

- **Testnet**は開発者が財務的リスクなしに実験できるAptosテストネットワークです。
- Testnetは通常mainnet機能を反映しますが、リセットされる場合があります。

### Transaction

- **トランザクション**はAptosブロックチェーンの状態を変更するために送信される操作です。
- トランザクションにはトークン転送、コントラクト展開などが含まれる場合があります。

## V

### Validator (バリデーター)

- **バリデーター**はコンセンサスに参加しネットワークセキュリティを維持するノードです。
- バリデーターはAPTトークンをステーキングし、プロトコルルールに従って報酬を受け取る必要があります。
