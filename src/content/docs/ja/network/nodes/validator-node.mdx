---
title: "バリデーターとVFNを実行"
---

Aptosのコンセンサスプロトコルに参加するには,バリデーターノードと(オプションで)バリデーターフルノード(VFN)をデプロイして実行する必要があります.このドキュメントでは,両方のノードタイプをデプロイするために必要な重要なステップの概要を説明します.

import { Aside, Steps } from '@astrojs/starlight/components';

<Aside type="note">
  VFNはコンセンサスに参加するために必須ではありませんが,すべてのバリデーター運営者がVFNを実行することを強く推奨します.
  これは,VFNがブロックチェーンデータのエコシステムにおける唯一の入出力ポイントだからです.ネットワーク内に多くのVFNがあることで,信頼性の向上,データ可用性の向上,一般ユーザーへの高品質なブロックチェーンアクセスの提供に役立ちます.
</Aside>

ステーキングプールまたはデリゲーションプールを初期化する前に,[ステーキング](/ja/network/blockchain/staking)と[デリゲートステーキング](/ja/network/blockchain/delegated-staking)について読み,ステークプールタイプの違いを理解してください.ステークプールが作成されると,デリゲーションプールに変更することはできず,その逆も同様です.

このセクションのドキュメントを使用して,Aptosのバリデーターとバリデーターフルノードを実行してください.概要として,プロセスは以下の通りです:

<Steps>
  1. 要件とデプロイメントタイプを理解する

     ノードの要件を読み,デプロイメント方法(オンプレミスまたはクラウドサービスなど)を選択します.
     まず,必要なコンピューティング,メモリ,ネットワーク,ストレージリソースを知るためにノードの要件を読んでください.
     また,クラウド管理のKubernetes,Docker,またはソースコードを使用するなど,ノードをデプロイする方法を選択してください.

  2. ノードのアイデンティティを生成する

     バリデーターとVFNの公開鍵/秘密鍵のペアとアカウントアドレスを作成します.
     秘密鍵は機密情報として保管してください!

  3. バリデーターとVFNを設定する

     生成したキーとアイデンティティを使用してノードを設定します.
     YAMLファイルを使用して,前のステップで生成したキーとアイデンティティでノードを設定します.
     これは,ノードが他のノード(ピア)と安全に接続できるようにするために必要です.

  4. ノードのgenesisとwaypointをダウンロードする

     genesisとwaypointでノードをブートストラップし,起動の準備をします.
     ノードが正しく設定されたら,バイナリをインストールし,genesisブロブとwaypointファイルをダウンロードします.
     これらにより,ノードは他のピアとの同期を開始するために必要な情報を得ることができます.

  5. バリデーターセットに参加して検証を開始する

     ステーキングプールを初期化し,バリデーターセットに参加して,報酬を得るための検証を開始します.
     他のピアがノードからの接続を受け入れる前に,バリデーターセットに参加する必要があります.
     そのためには,ステーキングプールを初期化し,オペレーターと投票者にデリゲートする必要があります.
     ステーキングプールの設定が完了したら,バリデーターセットに参加できます.
     この時点でノードはネットワークとの同期を開始し,バリデーターはコンセンサスに参加できるようになります.
     これが報酬を得始めることができるタイミングです.
</Steps>
