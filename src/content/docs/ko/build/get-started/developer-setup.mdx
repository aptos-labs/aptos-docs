개발 유형에 따라 환경을 설정하는 쉬운 방법입니다.

<Tabs>
  {/* Frontend */}

  <TabItem label="프론트엔드">
    <Steps>
      1. 프론트엔드 프로젝트 초기화

         다음은 인기 있는 선택지들의 예제입니다:

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. @aptos-labs/ts-sdk 설치

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. TS SDK 설정

         <CardGrid>
           <LinkCard href="/ko/build/sdks/ts-sdk/quickstart" title="TS SDK 퀵스타트" description="계정 설정, 네트워크, 파우셋 사용, 거래 전송/시뮬레이션 등의 방법을 확인하세요" />
         </CardGrid>

      4. 앱을 구축하세요!

         프론트엔드에서 Aptos를 사용하기 위한 개발자 설정이 완료되었습니다.
         개발 과정을 간소화하는 다른 도구들을 확인해보세요

         <CardGrid>
           <LinkCard href="/ko/build/indexer" title="인덱서" description="잔액, 거래 활동, 토큰 데이터 등의 온체인 상태를 효율적으로 쿼리하세요" />

           <LinkCard href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript" title="TS SDK 예제" description="TS SDK 사용 방법에 대한 20개 이상의 예제" target="_blank" />

           <LinkCard href="https://build.aptoslabs.com/" title="Aptos Build" description="Fullnode API / 인덱서의 속도 제한에 걸렸나요? 여기서 API 키를 받으세요" target="_blank" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Smart Contract */}

  <TabItem label="스마트 컨트랙트">
    <Steps>
      1. CLI 설치

         <CardGrid>
           <LinkCard href="/ko/build/cli" title="Aptos CLI" description="Aptos CLI 설치 방법에 대한 지침" />
         </CardGrid>

      2. 에디터 또는 IDE 설정

         Move 개발을 더 쉽게 만들기 위해 선택한 에디터에 다음 확장 프로그램을 추가하세요

         <Tabs>
           <TabItem label="JetBrains IDEs">
             <CardGrid>
               <LinkCard href="https://plugins.jetbrains.com/plugin/14721-move-language" title="Move on Aptos" description="CLion, Rust Rover, WebStorm, IntelliJ과 같은 JetBrains IDE용 언어 서버 및 구문 하이라이터" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=AptosLabs.move-on-aptos" title="Move on Aptos 확장" description="VSCode용 완전한 기능의 Aptos Move IDE" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="Cursor, Kiro 등">
             <CardGrid>
               <LinkCard href="https://open-vsx.org/extension/aptoslabs/move-on-aptos" title="Move on Aptos 확장" description="OpenVSX 기반 플랫폼용 완전한 기능의 Aptos Move IDE" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. 스마트 컨트랙트 생성

         애플리케이션 폴더로 이동하고 다음을 실행하여 새로운 스마트 컨트랙트를 초기화하세요:

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. 스마트 컨트랙트 빌드, 컴파일, 배포!

         스마트 컨트랙트를 위한 Aptos 사용 개발자 설정이 완료되었습니다.
         더 많은 정보는 아래 Dapp 튜토리얼 링크를 참조하세요

         <CardGrid>
           <LinkCard href="/ko/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file" title="스마트 컨트랙트 생성 가이드" description="Move를 사용하여 스마트 컨트랙트를 설정하는 방법에 대한 쉬운 할 일 목록 가이드" />
         </CardGrid>
    </Steps>
  </TabItem>

  <TabItem label="Create Aptos Dapp">

           <TabItem label="npx">

         CLI 프롬프트를 따라 새 dApp의 이름, [템플릿](/ko/build/create-aptos-dapp#templates), 네트워크를 선택하세요.

      3. 새 dApp 구축 및 사용자 정의 시작!

         다음 단계는 생성된 `README.md` 파일을 따르세요.

      4. 계속 읽기
