## A

### Accumulator Root Hash

- Um **accumulator root hash** é o hash raiz de um [acumulador Merkle](https://eprint.iacr.org/2009/625.pdf).

### Account (Conta)

- Uma **conta** na blockchain Aptos é um contêiner para um número arbitrário de [módulos Move](#move-module) e [recursos Move](#move-resources).
- O estado de cada conta é composto de código e dados.
- A conta é identificada pelo [endereço da conta](#account-address).

Veja [Contas](/network/blockchain/accounts) para mais informações.

### Account Address (Endereço da Conta)

- Um **endereço de conta** é o endereço de uma conta Aptos.
- O endereço da conta refere-se a um destino específico na rede Aptos. O endereço dita o destino e fonte de um valor específico de ativos trocados por duas partes na blockchain.
- Endereços Aptos são strings hexadecimais de 64 caracteres (32 bytes). Frequentemente essas strings são prefixadas com `0x` e para os primeiros 16 endereços, os zeros à esquerda são excluídos (ex. `0x1`).

### API

- Uma **Application Programming Interface (API)** é um conjunto de protocolos e ferramentas que permitem aos usuários interagir com nós da blockchain Aptos e redes cliente via aplicações externas. Aptos oferece uma API REST para se comunicar com nossos nós.
- Veja [documentação](/build/apis) para mais detalhes.

### APT

O **token Aptos (APT)** é o token nativo da blockchain Aptos usado para pagar taxas de rede e transação.

### Aptos

**Aptos** é uma blockchain Layer 1 para todos. Ela usa a linguagem de programação Move e lançou seu mainnet em 17-10-2022 para redefinir a experiência do usuário web3. A blockchain Aptos é dedicada a criar melhores experiências de usuário através de velocidade aumentada, segurança, escalabilidade, confiabilidade e usabilidade com custos de transação baixos. A palavra "Aptos" significa "As Pessoas" na linguagem Ohlone. Saiba mais sobre a blockchain Aptos no [site oficial Aptos](https://aptosfoundation.org).

### AptosBFT

- **AptosBFT** é o algoritmo de consenso BFT do protocolo Aptos.
- AptosBFT é baseado em Jolteon.

### Aptos Blockchain

- A **blockchain Aptos** é um ledger de transações imutáveis acordadas pelos validadores na rede Aptos (a rede de validadores).

### Aptos Name Service (ANS)

- O **Aptos Name Service (ANS)** é um serviço de endereçamento de nomes descentralizado para a blockchain Aptos. Um nome Aptos é um nome de domínio _.apt_ legível por humanos usado no lugar de uma chave pública, por exemplo _love.apt_.
- Este serviço também permite aos usuários registrar nomes de subdomínio além do domínio registrado. Descubra mais em: [Aptosnames.com](https://www.aptosnames.com/)

### Aptos Framework

O **Aptos Framework** define a API pública para atualizações da blockchain e a estrutura de dados on-chain. Ele define a lógica de negócio e controle de acesso para os três pilares chave da funcionalidade Aptos: pagamentos, tesouraria e governança on-chain. Ele é implementado como um conjunto de módulos escritos na linguagem de programação Move e armazenados on-chain como bytecode Move.

### Aptos Node

Um **nó Aptos** é uma entidade peer da rede Aptos que rastreia o estado da blockchain Aptos. Há dois tipos de nós Aptos: [validadores](#validator) e [nós completos](#fullnodes).

## B

### Blocks (Blocos)

- No Aptos, blocos são um lote de [transações](#transaction) confirmadas ao mesmo tempo.
- O número do bloco é análogo a "altura do bloco" na literatura blockchain.

### Byzantine Fault Tolerance (BFT)

- **Byzantine Fault Tolerance (BFT)** é a propriedade de um sistema distribuído de resistir a falhas bizantinas (falhas arbitrárias).
- BFT garante que um sistema continuará funcionando corretamente mesmo se alguns nós falharem ou agirem maliciosamente.

## C

### Consensus

- O **consenso** é o mecanismo pelo qual nós de uma rede distribuída chegam a um acordo sobre o estado da blockchain.
- Aptos usa AptosBFT como algoritmo de consenso.

## D

### Devnet

- **Devnet** é a rede de desenvolvimento Aptos onde desenvolvedores podem testar suas aplicações e contratos inteligentes.
- Devnet é atualizado regularmente com os últimos recursos mas pode ser reiniciado sem aviso.

## E

### Events (Eventos)

- **Eventos** são mensagens emitidas por contratos inteligentes Move durante a execução.
- Eventos permitem que aplicações off-chain reajam a mudanças de estado on-chain.

## F

### Fullnodes (Nós Completos)

- Um **nó completo (fullnode)** é um nó que mantém uma cópia completa do estado da blockchain.
- Ao contrário dos validadores, fullnodes não participam do consenso mas podem validar transações.

## G

### Gas

- **Gás** é a unidade que mede o custo computacional das transações no Aptos.
- Gás é pago em APT e cobre execução, armazenamento e taxas de rede.

### Governance (Governança)

- **Governança** refere-se ao processo pelo qual detentores de tokens podem propor e votar sobre mudanças do protocolo Aptos.
- Governança on-chain permite que detentores de APT participem diretamente das decisões da rede.

## M

### Mainnet

- **Mainnet** é a rede principal de produção do Aptos onde transações têm valor real.
- Mainnet é a rede oficial e estável usada por usuários finais e aplicações de produção.

### Merkle Trees

- **Árvores Merkle** são estruturas de dados usadas para verificar eficientemente a integridade dos dados em blockchains.
- Elas permitem verificar que um elemento faz parte de um conjunto maior sem revelar o conjunto completo.

### Move

- **Move** é a linguagem de programação de contratos inteligentes projetada especificamente para Aptos.
- Move enfatiza segurança, modularidade e verificação formal.

### Move Module

- Um **módulo Move** é uma unidade de código implantada na blockchain Aptos.
- Módulos contêm definições de tipos, funções e recursos.

### Move Resources

- **Recursos Move** são instâncias de estruturas que armazenam dados na blockchain.
- Ao contrário dos módulos, recursos contêm dados e não código.

## S

### Staking

- **Staking** é o processo de travar tokens APT para apoiar um validador e ganhar recompensas.
- Detentores de APT podem apostar seus tokens para ajudar a proteger a rede e ganhar recompensas.

## T

### Testnet

- **Testnet** é a rede de teste Aptos onde desenvolvedores podem experimentar sem risco financeiro.
- Testnet geralmente reflete recursos do mainnet mas pode ser reiniciado.

### Transaction

- Uma **transação** é uma operação submetida à blockchain Aptos para mudar seu estado.
- Transações podem incluir transferências de tokens, implantações de contratos, etc.

## V

### Validator (Validador)

- Um **validador** é um nó que participa do consenso e ajuda a manter a segurança da rede.
- Validadores devem apostar tokens APT e seguir regras do protocolo para ganhar recompensas.
