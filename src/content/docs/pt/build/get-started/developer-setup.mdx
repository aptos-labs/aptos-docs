import { CardGrid, LinkCard, Steps, TabItem, Tabs } from '@astrojs/starlight/components';

Aqui está uma maneira fácil de configurar seu ambiente dependendo do tipo de desenvolvimento.

<Tabs>
  {/* Frontend */}

  <TabItem label="Frontend">
    <Steps>
      1. Inicializar Projeto Frontend

         Aqui estão alguns exemplos de escolhas populares:

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. Instalar @aptos-labs/ts-sdk

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. Configurar TS SDK

         <CardGrid>
           <LinkCard
             href="/build/sdks/ts-sdk/quickstart"
             title="Início Rápido TS SDK"
             description="Veja como configurar sua conta, rede, usar o
         faucet, enviar / simular transações, e mais"
           />
         </CardGrid>

      4. Construa seu app!

         A configuração do desenvolvedor para usar Aptos em seu frontend está agora completa.
         Confira nossas outras ferramentas que simplificam o processo de desenvolvimento

         <CardGrid>
           <LinkCard
             href="/build/indexer"
             title="Indexador"
             description="Consulte eficientemente o estado on-chain como saldos,
         atividade de transação, dados de token, e mais"
           />

           <LinkCard
             href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript"
             title="Exemplos TS SDK"
             description="20+ Exemplos de como usar o TS
         SDK"
             target="_blank"
           />

           <LinkCard href="https://geomi.dev/" title="Geomi" description="Atingindo limites de taxa para Fullnode API / Indexadores? Obtenha uma chave API aqui" target="_blank" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Smart Contract */}

  <TabItem label="Contrato Inteligente">
    <Steps>
      1. Instalar CLI

         <CardGrid>
           <LinkCard href="/build/cli" title="Aptos CLI" description="Instruções para instalar Aptos CLI" />
         </CardGrid>

      2. Configurar Editor ou IDE

         Adicione as seguintes extensões ao seu editor de escolha para tornar o desenvolvimento Move mais fácil

         <Tabs>
           <TabItem label="IDEs JetBrains">
             <CardGrid>
               <LinkCard
                 href="https://plugins.jetbrains.com/plugin/14721-move-language"
                 title="Move on Aptos"
                 description="Servidor de linguagem e destacador de sintaxe para
         IDEs JetBrains como CLion, Rust Rover, WebStorm,
         IntelliJ"
                 target="_blank"
               />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=AptosLabs.move-on-aptos" title="Extensão Move on Aptos" description="IDE Aptos Move completo para VSCode" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="Cursor, Kiro, etc.">
             <CardGrid>
               <LinkCard href="https://open-vsx.org/extension/aptoslabs/move-on-aptos" title="Extensão Move on Aptos" description="IDE Aptos Move completo para plataformas OpenVSX" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. Criar Contrato Inteligente

         Navegue até sua pasta de aplicação e inicialize um novo contrato inteligente fazendo:

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. Construir, Compilar e Implantar Contrato Inteligente!

         A configuração do desenvolvedor para usar Aptos para contratos inteligentes está agora completa.
         Para mais informações, veja o link para o tutorial Dapp abaixo

         <CardGrid>
           <LinkCard
             href="/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file"
             title="Guia Criação Contrato Inteligente"
             description="Um guia fácil de lista de tarefas para configurar um contrato
         inteligente com Move"
           />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* create-aptos-dapp */}

  <TabItem label="Criar DApp Aptos">
    <Steps>
      1. Instalar create-aptos-dapp

         Execute o comando abaixo para instalar uma dApp a partir de um modelo em segundos:

         <Tabs>
           { /* npx */ }

           <TabItem label="npx">
             ```shellscript filename="Terminal"
             npx create-aptos-dapp@latest
             ```
           </TabItem>

           { /* pnpx */ }

           <TabItem label="pnpx">
             ```shellscript filename="Terminal"
             pnpx create-aptos-dapp@latest
             ```
           </TabItem>
         </Tabs>

      2. Siga as instruções

         Siga as instruções CLI para selecionar um nome, [modelo](/build/create-aptos-dapp#templates), e rede para sua nova dApp.

         ![cad](~/images/cad-video.gif)

      3. Comece a construir e personalizar sua nova dApp!

         Navegue até seu novo projeto e abra em seu IDE favorito para continuar construindo!

         Siga o arquivo `README.md` gerado para os próximos passos.

      4. Continue lendo

         <CardGrid>
           <LinkCard href="/build/create-aptos-dapp" title="Criar DApp Aptos" description="Obtenha mais informações sobre a ferramenta" />

           <LinkCard href="/build/create-aptos-dapp#templates" title="Modelos" description="Navegue por modelos pré-fabricados" />

           <LinkCard href="/build/create-aptos-dapp/faq" title="FAQ" description="Obtenha ajuda para problemas e questões comuns" />
         </CardGrid>
    </Steps>
  </TabItem>
</Tabs>
