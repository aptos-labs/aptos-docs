Вот простой способ настроить среду в зависимости от типа разработки.

<Tabs>
  {/* Frontend */}

  <TabItem label="Фронтенд">
    <Steps>
      1. Инициализация фронтенд-проекта

         Вот несколько примеров популярных вариантов:

         <Tabs>
           <TabItem label="Next.js">
             ```shellscript filename="Terminal"
             pnpx create-next-app@latest
             ```
           </TabItem>

           <TabItem label="Vite (TS)">
             ```shellscript filename="Terminal"
             pnpx create vite my-aptos-app --template react-ts
             ```
           </TabItem>
         </Tabs>

      2. Установка @aptos-labs/ts-sdk

         ```shellscript npm2yarn
         npm i @aptos-labs/ts-sdk
         ```

      3. Настройка TS SDK

         <CardGrid>
           <LinkCard
             href="/build/sdks/ts-sdk/quickstart"
             title="Быстрый старт TS SDK"
             description="Узнайте, как настроить аккаунт, сеть, использовать
         faucet, отправлять / симулировать транзакции и многое другое"
           />
         </CardGrid>

      4. Создайте свое приложение!

         Настройка разработчика для использования Aptos во фронтенде теперь завершена.
         Ознакомьтесь с другими нашими инструментами, которые упрощают процесс разработки

         <CardGrid>
           <LinkCard
             href="/build/indexer"
             title="Индексатор"
             description="Эффективно запрашивайте состояние ончейн, такое как балансы,
         активность транзакций, данные токенов и многое другое"
           />

           <LinkCard
             href="https://github.com/aptos-labs/aptos-ts-sdk/tree/main/examples/typescript"
             title="Примеры TS SDK"
             description="20+ примеров использования
         TS SDK"
             target="_blank"
           />

           <LinkCard href="https://geomi.dev/" title="Geomi" description="Достигли лимитов скорости для Fullnode API / Индексаторов? Получите API-ключ здесь" target="_blank" />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* Smart Contract */}

  <TabItem label="Смарт-контракт">
    <Steps>
      1. Установка CLI

         <CardGrid>
           <LinkCard href="/build/cli" title="Aptos CLI" description="Инструкции по установке Aptos CLI" />
         </CardGrid>

      2. Настройка редактора или IDE

         Добавьте следующие расширения в выбранный вами редактор для упрощения разработки Move

         <Tabs>
           <TabItem label="IDEs JetBrains">
             <CardGrid>
               <LinkCard
                 href="https://plugins.jetbrains.com/plugin/14721-move-language"
                 title="Move on Aptos"
                 description="Сервер языка и подсветка синтаксиса для
         JetBrains IDEs, таких как CLion, Rust Rover,
         WebStorm, IntelliJ"
                 target="_blank"
               />
             </CardGrid>
           </TabItem>

           <TabItem label="VSCode">
             <CardGrid>
               <LinkCard href="https://marketplace.visualstudio.com/items?itemName=AptosLabs.move-on-aptos" title="Расширение Move on Aptos" description="Полнофункциональная Aptos Move IDE для VSCode" target="_blank" />
             </CardGrid>
           </TabItem>

           <TabItem label="Cursor, Kiro и т.д.">
             <CardGrid>
               <LinkCard href="https://open-vsx.org/extension/aptoslabs/move-on-aptos" title="Расширение Move on Aptos" description="Полнофункциональная Aptos Move IDE для платформ OpenVSX" target="_blank" />
             </CardGrid>
           </TabItem>
         </Tabs>

      3. Создание смарт-контракта

         Перейдите в папку вашего приложения и инициализируйте новый смарт-контракт, выполнив:

         ```shellscript filename="Terminal"
         aptos move init --name my_todo_list
         ```

      4. Сборка, компиляция и развертывание смарт-контракта!

         Настройка разработчика для использования Aptos для смарт-контрактов теперь завершена.
         Для дополнительной информации см. ссылку на учебник Dapp ниже

         <CardGrid>
           <LinkCard
             href="/build/guides/build-e2e-dapp/1-create-smart-contract#what-is-a-movetoml-file"
             title="Руководство по созданию смарт-контракта"
             description="Простое руководство по списку задач для настройки смарт-
         контракта с Move"
           />
         </CardGrid>
    </Steps>
  </TabItem>

  {/* create-aptos-dapp */}

  <TabItem label="Создание Aptos Dapp">
    <Steps>
      1. Установка create-aptos-dapp

         Чтобы установить dApp из шаблона за секунды, выполните команду ниже:

         <Tabs>
           { /* npx */ }

           <TabItem label="npx">
             ```shellscript filename="Terminal"
             npx create-aptos-dapp@latest
             ```
           </TabItem>

           { /* pnpx */ }

           <TabItem label="pnpx">
             ```shellscript filename="Terminal"
             pnpx create-aptos-dapp@latest
             ```
           </TabItem>
         </Tabs>

      2. Следуйте подсказкам

         Следуйте подсказкам CLI, чтобы выбрать имя, [шаблон](/build/create-aptos-dapp#templates) и сеть для вашего нового dApp.

         ![cad](~/images/cad-video.gif)

      3. Начните создавать и настраивать свое новое dApp!

         Перейдите в новый проект и откройте в предпочитаемой IDE, чтобы продолжить разработку!

         Следуйте созданному файлу `README.md` для следующих шагов.

      4. Продолжайте читать

           <LinkCard href="/build/create-aptos-dapp" title="Создание Aptos Dapp" description="Получите дополнительную информацию об инструменте" />

           <LinkCard href="/build/create-aptos-dapp/faq" title="FAQ" description="Получите помощь по распространенным проблемам и вопросам" />
         </CardGrid>
    </Steps>
  </TabItem>
