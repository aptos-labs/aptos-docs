## А

### Accumulator Root Hash

- **accumulator root hash** — это корневой хэш [накопителя Merkle](https://eprint.iacr.org/2009/625.pdf).

### Account (Аккаунт)

- **Аккаунт** в блокчейне Aptos — это контейнер для произвольного количества [модулей Move](#move-module) и [ресурсов Move](#move-resources).
- Состояние каждого аккаунта состоит из кода и данных.
- Аккаунт идентифицируется [адресом аккаунта](#account-address).

См. [Аккаунты](/network/blockchain/accounts) для дополнительной информации.

### Account Address (Адрес аккаунта)

- **Адрес аккаунта** — это адрес аккаунта Aptos.
- Адрес аккаунта относится к конкретному назначению в сети Aptos. Адрес определяет назначение и источник определенной суммы активов, обмениваемых двумя сторонами в блокчейне.
- Адреса Aptos — это 64-символьные шестнадцатеричные строки (32 байта). Часто эти строки предваряются `0x`, и для первых 16 адресов ведущие нули опускаются (пример: `0x1`).

### API

- **API (Application Programming Interface)** — это набор протоколов и инструментов, которые позволяют пользователям взаимодействовать с узлами блокчейна Aptos и клиентскими сетями через внешние приложения. Aptos предоставляет REST API для общения с нашими узлами.
- См. [документацию](/build/apis) для деталей.

### APT

**Токен Aptos (APT)** — это нативный токен блокчейна Aptos, используемый для оплаты сетевых и транзакционных сборов.

### Aptos

**Aptos** — это блокчейн Layer 1 для всех. Он использует язык программирования Move и запустил свой mainnet 17-10-2022 для переопределения пользовательского опыта web3. Блокчейн Aptos посвящен созданию лучших пользовательских опытов благодаря увеличенной скорости, безопасности, масштабируемости, надежности и удобству использования с низкими затратами на транзакции. Слово "Aptos" означает "The People" на языке Ohlone. Узнайте больше о блокчейне Aptos на [официальном сайте Aptos](https://aptosfoundation.org).

### AptosBFT

- **AptosBFT** — это алгоритм консенсуса BFT протокола Aptos.
- AptosBFT основан на Jolteon.

### Aptos Blockchain

- **Блокчейн Aptos** — это реестр неизменяемых транзакций, согласованных валидаторами в сети Aptos (сети валидаторов).

### Aptos Name Service (ANS)

- **Служба имен Aptos (ANS)** — это децентрализованная служба адресации имен для блокчейна Aptos. Имя Aptos — это доменное имя _.apt_, читаемое человеком, используемое вместо публичного ключа, например _love.apt_.
- Этот сервис также позволяет пользователям регистрировать субдоменные имена в дополнение к зарегистрированному домену. Узнайте больше на [Aptosnames.com](https://www.aptosnames.com).

### Aptos Framework

**Фреймворк Aptos** определяет публичный API для обновлений блокчейна и структуру данных в цепочке. Он определяет бизнес-логику и контроль доступа для трех ключевых столпов функциональности Aptos: платежи, казначейство и ончейн-управление. Он реализован как набор модулей, написанных на языке программирования Move, и хранится в цепочке как байткод Move.

### Aptos Node

**Узел Aptos** — это одноранговый объект в сети Aptos, отслеживающий состояние блокчейна Aptos. Существует два типа узлов Aptos: [валидаторы](#validator) и [полные узлы](#fullnodes).

## Б

### Blocks (Блоки)

- В Aptos блоки — это партия [транзакций](#transaction), зафиксированных одновременно.
- Номер блока аналогичен "высоте блока" в литературе по блокчейну.

### Byzantine Fault Tolerance (BFT)

- **Byzantine Fault Tolerance (BFT)** — это свойство распределенной системы, позволяющее выдерживать византийские сбои (произвольные сбои).
- BFT гарантирует, что система будет продолжать работать правильно, даже если некоторые узлы выйдут из строя или будут действовать вредоносно.

## К

### Consensus

- **Консенсус** — это механизм, с помощью которого узлы распределенной сети приходят к согласию относительно состояния блокчейна.
- Aptos использует AptosBFT в качестве алгоритма консенсуса.

## Д

### Devnet

- **Devnet** — это сеть разработки Aptos, где разработчики могут тестировать свои приложения и смарт-контракты.
- Devnet регулярно обновляется новейшими функциями, но может быть сброшен без предупреждения.

## С

### Events (События)

- **События** — это сообщения, испускаемые смарт-контрактами Move во время выполнения.
- События позволяют офчейн-приложениям реагировать на изменения состояния в цепочке.

## П

### Fullnodes (Полные узлы)

- **Полный узел (fullnode)** — это узел, поддерживающий полную копию состояния блокчейна.
- В отличие от валидаторов, полные узлы не участвуют в консенсусе, но могут проверять транзакции.

## Г

### Gas

- **Газ** — это единица измерения вычислительной стоимости транзакций в Aptos.
- Газ оплачивается в APT и покрывает выполнение, хранение и сетевые сборы.

### Governance (Управление)

- **Управление** относится к процессу, с помощью которого держатели токенов APT могут предлагать и голосовать за изменения протокола Aptos.
- Ончейн-управление позволяет держателям APT напрямую участвовать в решениях сети.

## М

### Mainnet

- **Mainnet** — это основная производственная сеть Aptos, где транзакции имеют реальную ценность.
- Mainnet — это официальная и стабильная сеть, используемая конечными пользователями и производственными приложениями.

### Merkle Trees

- **Деревья Merkle** — это структуры данных, используемые для эффективной проверки целостности данных в блокчейнах.
- Они позволяют проверять, что элемент является частью большего набора, не раскрывая весь набор.

### Move

- **Move** — это язык программирования смарт-контрактов, специально разработанный для Aptos.
- Move акцентирует безопасность, модульность и формальную верификацию.

### Move Module

- **Модуль Move** — это единица кода, развернутая в блокчейне Aptos.
- Модули содержат определения типов, функций и ресурсов.

### Move Resources

- **Ресурсы Move** — это экземпляры структур, хранящих данные в блокчейне.
- В отличие от модулей, ресурсы содержат данные, а не код.

## Р

### Resources (Ресурсы)

- См. [Ресурсы Move](#move-resources).

## С

### Staking

- **Стейкинг** — это процесс блокировки токенов APT для поддержки валидатора и получения вознаграждений.
- Держатели APT могут стейкать свои токены для помощи в обеспечении безопасности сети и получения вознаграждений.

## Т

### Testnet

- **Testnet** — это тестовая сеть Aptos, где разработчики могут экспериментировать без финансового риска.
- Testnet обычно отражает функции mainnet, но может быть сброшен.

### Transaction

- **Транзакция** — это операция, отправленная в блокчейн Aptos для изменения его состояния.
- Транзакции могут включать переводы токенов, развертывание контрактов и т.д.

## В

### Validator (Валидатор)

- **Валидатор** — это узел, участвующий в консенсусе и помогающий поддерживать безопасность сети.
- Валидаторы должны стейкать токены APT и следовать правилам протокола для получения вознаграждений.
