Aptos 为 AI 驱动的开发工作流提供一流支持。无论您使用的是 AI 编程助手还是 AI 聊天工具，
都可以让它深入了解 Aptos 区块链、Move 语言、SDK 和 API。

## AskAptos 聊天机器人

AskAptos 聊天机器人已内置于文档站点中——在导航栏右上角查找即可。它可以回答关于 Aptos 概念的问题，帮助您规划实现方案，并指引您到正确的文档页面。

## Aptos MCP 服务器

[模型上下文协议 (MCP)](https://modelcontextprotocol.io/) 让 AI 编程工具能够直接调用 Aptos API、
查询链上数据并生成正确的 Aptos 代码，无需复制粘贴文档。

## Aptos Agent Skills

[Aptos Agent Skills](https://github.com/aptos-labs/aptos-agent-skills) 提供预构建 AI 技能，
为编程助手提供深度 Aptos 专业知识——涵盖 Move 智能合约、TypeScript SDK 和全栈 dApp 脚手架。

```bash
npx skills add aptos-labs/aptos-agent-skills
```

此命令使用通过 `npx` 调用的 [Skills CLI](https://www.npmjs.com/package/skills)。如果当前环境中没有 `npx skills`，请确认已安装 Node.js 与 npm，并参考 Skills CLI 文档完成安装与排查。

## LLMs.txt 文档源

[LLMs.txt 标准](https://llmstxt.org/)提供机器可读的文档源，AI 工具可以摄取这些内容以理解完整的 Aptos
文档。我们发布三种文档源以适应不同的上下文窗口大小：

| 文档源                | URL                                  | 最适合                         |
| ------------------ | ------------------------------------ | --------------------------- |
| **llms.txt**       | [`/llms.txt`](/llms.txt)             | 包含页面标题、描述和逐页 `.md` 链接的结构化索引 |
| **llms-small.txt** | [`/llms-small.txt`](/llms-small.txt) | 适合较小上下文窗口的压缩文档              |
| **llms-full.txt**  | [`/llms-full.txt`](/llms-full.txt)   | 完整文档——所有页面串联在一起             |

部分客户端会解析 `/.well-known/llms.txt`；生产环境会通过重定向提供与 [`/llms.txt`](/llms.txt) 相同的索引（[`https://aptos.dev/.well-known/llms.txt`](https://aptos.dev/.well-known/llms.txt)）。

[`llms.txt`](/llms.txt) 路由还会列出机器可读的 **API** 链接——[`/aptos-spec.json`](/aptos-spec.json)（全节点 REST API 的 OpenAPI 3 JSON）以及 [`/zh/rest-api`](/zh/rest-api)（HTML 参考文档）——并在同一处汇总 **MCP**、Agent Skills、浏览器、GitHub、标准与 Indexer GraphQL 等 `.md` 链接。

## 面向智能体的发现端点

为方便编程智能体一站式发现 Aptos 的机器可读资源，文档站点发布了一组标准化的发现文档。每个页面的响应也会带上 [`Link`](https://www.rfc-editor.org/rfc/rfc8288) 响应头与对应的 `<link rel="…">` 标签，指向相同的 URL。

| 端点                                                                                   | 作用                                                                                                                                                                                        |
| ------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`/.well-known/api-catalog`](/.well-known/api-catalog)                               | 符合 [RFC 9727](https://www.rfc-editor.org/rfc/rfc9727) 的 linkset，指向 REST API 规范、文档、Indexer GraphQL 以及 LLMs.txt 文档源。                                                                        |
| [`/.well-known/mcp/server-card.json`](/.well-known/mcp/server-card.json)             | 描述 [`@aptos-labs/aptos-mcp`](https://www.npmjs.com/package/@aptos-labs/aptos-mcp) 的 [MCP Server Card (SEP-2127)](https://github.com/modelcontextprotocol/modelcontextprotocol/pull/2127)。 |
| [`/.well-known/agent-skills/index.json`](/.well-known/agent-skills/index.json)       | 基于 [Agent Skills Discovery](https://github.com/cloudflare/agent-skills-discovery-rfc) 的 [aptos-agent-skills](https://github.com/aptos-labs/aptos-agent-skills) 索引，包含每个技能的 SHA-256 摘要。     |
| [`/.well-known/oauth-protected-resource`](/.well-known/oauth-protected-resource)     | 面向 Google 登录的 [Aptos 测试网水龙头](/zh/network/faucet) 的 [OAuth 2.0 受保护资源元数据（RFC 9728）](https://www.rfc-editor.org/rfc/rfc9728)，列出资源标识符、授权服务器以及支持的 scopes。                                      |
| [`/.well-known/openid-configuration`](/.well-known/openid-configuration)             | [OpenID Connect Discovery 1.0](https://openid.net/specs/openid-connect-discovery-1_0.html) 文档，描述 [Aptos 测试网水龙头](/zh/network/faucet) 登录所使用的 issuer、授权与令牌端点、JWKS URI 与支持的授权类型。              |
| [`/.well-known/oauth-authorization-server`](/.well-known/oauth-authorization-server) | 面向偏好纯 OAuth 2.0 端点名称的客户端的 [OAuth 2.0 授权服务器元数据（RFC 8414）](https://www.rfc-editor.org/rfc/rfc8414) 副本，与上面的 OIDC 发现文档内容一致。                                                                   |
| [`/robots.txt`](/robots.txt)                                                         | 包含 [Content Signals](https://contentsignals.org/)（`ai-train=yes, search=yes, ai-input=yes`），表明 Aptos 文档允许 AI 使用。                                                                          |

OAuth / OIDC 发现文档仅适用于 [Aptos 测试网水龙头](/zh/network/faucet)；`aptos.dev` 的其余部分都是公开的、无需鉴权的文档。需要为测试网账户充值的智能体可以调用 `/.well-known/oauth-protected-resource` 获取水龙头的资源标识符以及它在 `Authorization: Bearer …` 头中接受的 OIDC issuer（Firebase + Google）。

### Markdown 内容协商

任何拥有 `.md` 镜像的文档 URL（例如 [`/zh/build/sdks`](/zh/build/sdks) → [`/zh/build/sdks.md`](/zh/build/sdks.md)）都会响应 `Accept: text/markdown` 并返回 `Content-Type: text/markdown`：

```bash
curl -H "Accept: text/markdown" https://aptos.dev/zh/build/sdks
```

浏览器的默认 `Accept: text/html` 依旧得到 HTML 响应。

### WebMCP

在实现 [WebMCP 提案](https://webmachinelearning.github.io/webmcp/) 的浏览器（如 Chrome 早期预览）中，每个文档页面会通过 `navigator.modelContext` 暴露以下工具：`aptos-docs.search`、`aptos-docs.open-doc`、`aptos-docs.fetch-doc-markdown` 与 `aptos-docs.list-llms-feeds`。
