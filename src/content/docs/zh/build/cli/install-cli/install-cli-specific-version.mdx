title: "安装特定版本的 Aptos CLI（高级）"
---

import { Steps } from '@astrojs/starlight/components';

如果你需要特定版本的 Aptos CLI,你可以直接从 Aptos 源代码构建它.这种安装方法主要用于与 Devnet 上的特定功能交互,这些功能可能尚未进入 Testnet / Mainnet.如果你运行的架构与现有版本/预编译二进制文件不兼容,你可能也需要按照这些步骤操作.

如果你不需要这种高级方法,你可以在[这里](/zh/build/cli)找到普通的安装步骤.

## 在 macOS / Linux 上安装

<Steps>
  1. 按照 的步骤操作.

  2. 按照上的步骤确保已安装 cargo.

  3. 构建 CLI 工具:cargo build --package aptos --profile cli.

     二进制文件将位于 `target/cli/aptos`.

  4. (可选) 将此可执行文件移动到 PATH 中的某个位置.

  5. 通过运行 target/cli/aptos help 验证安装是否成功.

     这些帮助说明也可以作为特定命令的详细指南.
</Steps>

## 在 Windows 上安装

<Steps>
  1. 按照的步骤从源代码构建 Aptos.

  2. 按照上的步骤确保已安装 cargo.

  3. 构建 CLI 工具:cargo build --package aptos --profile cli.

     二进制文件将位于 `target\cli\aptos.exe`.

  4. (可选) 将此可执行文件移动到 PATH 中的某个位置.

  5. 通过运行 target\cli\aptos.exe help 验证安装是否成功.

     这些帮助说明也可以作为特定命令的详细指南.
</Steps>
