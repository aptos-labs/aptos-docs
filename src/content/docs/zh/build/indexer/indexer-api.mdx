Aptos Labs 托管了一个公共版本的索引器 GraphQL API，任何人都可以使用它来获取链上交易、可替代资产和代币的基本历史和聚合数据。

您可以通过查看下面您感兴趣的网络的 Hasura Explorer 来手动探索它。

您也可以通过下面的 GraphQL 端点访问 API。有关每个字段/表中数据格式的更多信息，请参阅[表参考页面](/zh/build/indexer/indexer-api/indexer-reference)。

## SDK 访问（主要方法）

使用索引器的主要方式是通过 [TypeScript SDK](/zh/build/sdks/ts-sdk/fetch-data-via-sdk) 访问它。

TypeScript SDK 将自动处理速率限制，并可以根据需要的数据无缝地允许 [Fullnode REST API](/zh/build/apis/fullnode-rest-api) 访问和索引器访问。

## GraphQL API 端点（直接访问）

如果您需要直接向 Aptos-Labs 托管的索引器 API 发出 GraphQL 查询，请使用以下端点：

- **主网：** `https://api.mainnet.aptoslabs.com/v1/graphql`
- **测试网：** `https://api.testnet.aptoslabs.com/v1/graphql`
- **开发网：** `https://api.devnet.aptoslabs.com/v1/graphql`

### 速率限制

通过阅读 [Aptos Build 文档](https://build.aptoslabs.com/docs/start/billing) 了解更多适用于 Aptos Labs 托管的索引器 API 的速率限制。

如果您需要更高的速率限制，请考虑以下解决方案：

1. 从 [Aptos Build](https://build.aptoslabs.com/) 获取 API 密钥。在 [Aptos Build 文档站点](https://build.aptoslabs.com/docs/start/api-keys) 了解更多关于 API 密钥的信息。
2. 自己运行 Aptos 索引器 API。参见[这里](/zh/build/indexer/txn-stream/self-hosted)的自托管指南。
